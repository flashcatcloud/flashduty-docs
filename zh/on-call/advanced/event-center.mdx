## 功能概述

事件中心提供了一个时间轴视角，帮助你将故障与变更事件在时间维度上进行关联分析。通过甘特图可视化，你可以直观地看到故障发生前后有哪些变更操作，从而快速定位故障是否与近期变更有关。

## 甘特图可视化

事件中心的核心是甘特图视图，它在同一条时间轴上同时展示故障和变更事件：

- **故障事件**：展示故障的触发和恢复时间段，以不同颜色区分严重程度
- **变更事件**：按变更来源（集成）分行展示，每个变更来源占据独立的一行

甘特图支持以下交互：

| 操作 | 说明 |
| :--- | :--- |
| **时间范围选择** | 通过右上角时间选择器调整查看范围，默认展示最近 6 小时 |
| **点击故障** | 点击甘特图上的故障条，右侧面板将展示故障详情摘要 |
| **鼠标悬浮** | 悬浮在时间轴上，显示对应时间点的参考线 |

## 筛选与过滤

事件中心提供筛选面板，帮助你聚焦到特定范围的事件：

- **标签筛选**：按告警和变更事件的标签进行过滤，支持多条件组合
- **变更来源筛选**：选择展示特定变更来源的事件

筛选条件支持通过 URL 参数传递，你可以将筛选结果的链接分享给团队成员。

## 典型使用场景

<AccordionGroup>
<Accordion title="故障根因分析">
当故障发生后，打开事件中心，查看故障触发时间点前后是否有变更事件发生。如果发现故障触发时间与某个部署变更高度吻合，该变更很可能是故障的根因。
</Accordion>
<Accordion title="变更影响评估">
在执行重大变更后，通过事件中心监控是否有新的故障触发。如果变更后短时间内出现大量故障，说明变更可能引入了问题，应考虑回滚。
</Accordion>
</AccordionGroup>
