## 创建复盘报告

当故障被关闭后，就可以开始进入复盘流程了。

<Steps>
<Step title="选择故障">
在故障列表或故障详情页中，选择需要复盘的故障。
</Step>

<Step title="选择模板">
选择一个复盘模板来确定报告的章节结构。系统提供中英文内置模板，你也可以选择团队创建的自定义模板。
</Step>

<Step title="一键创建">
点击“创建复盘报告”，系统会自动从故障中提取严重程度、处理时间、响应人员等信息，生成一份草稿状态的报告，并跳转到编辑器页面。
</Step>
</Steps>

<Note>
目前，同一个故障只能对应一份复盘报告。如果这些故障已经有了复盘报告，系统会直接打开已有的报告。
</Note>

## 在线协作编辑

打开报告后，你会进入一个在线协作编辑器。如果你用过 Google Docs、Notion 或飞书文档，会感到非常熟悉：多位团队成员可以同时编辑同一份报告，所有修改实时可见。

编辑器会自动保存你的修改，无需手动点击保存按钮。

### 插入图片

编辑过程中，你可以直接粘贴或拖拽图片到编辑器中：比如监控大盘截图、架构拓扑图或故障现场的关键信息。

| 限制项 | 说明 |
| :--- | :--- |
| 支持格式 | JPEG、PNG、WebP、GIF |
| 单张大小 | 最大 4 MB |
| 每份报告 | 最多 100 张 |

## 完善报告信息

除了报告正文，还有几项重要信息值得关注：

### 标题

系统会根据关联故障的标题自动生成一个默认标题。你可以直接在编辑器顶部修改为更精准的描述，比如"2026-03-15 支付服务超时导致订单失败"。标题最长支持 256 个字符。

### 基本信息

基本信息面板展示了这次故障的关键指标，你可以根据实际情况调整：

| 字段 | 说明 |
| :--- | :--- |
| 最高严重程度 | 这次故障最严重到什么程度 |
| 最早开始时间 | 故障最早触发的时间 |
| 最晚关闭时间 | 故障最晚关闭的时间 |
| 总持续时间 | 所有关联故障的持续时间之和 |
| 响应人员 | 参与故障处理的人员 |

### 跟进事项

跟进事项用来记录后续的改进行动，比如“优化数据库连接池配置”或“增加缓存熔断机制”。跟进事项在草稿和已发布状态下都可以随时编辑更新。

## 发布与管理

### 发布报告

当团队确认内容无误后，点击“发布”按钮将报告状态从草稿切换为已发布。

<Tip>
发布不是编写的终点。如果后续有新的发现或需要修正，切回草稿修改后重新发布即可。
</Tip>

### 查找报告

在复盘报告列表中，你可以按状态、团队、协作空间和创建时间范围来筛选报告，支持按创建时间或更新时间排序。

### 删除报告

如果一份报告确实不再需要，可以在报告详情页点击删除。

<Warning>
删除操作不可恢复，请谨慎操作。
</Warning>
