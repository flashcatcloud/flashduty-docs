## 为什么需要模板

每次复盘都要从零开始思考“应该写哪些内容”，既费时又容易遗漏关键信息。模板帮你解决这个问题：预先定义好章节结构和填写指引，让每份复盘报告都遵循统一的规范，也让 AI 生成内容时有章可循。

<Note>
故障复盘功能仅在专业版中提供。
</Note>

## 内置模板

系统开箱即用地提供了两个内置模板：

| 模板 | 语言 | 包含章节 |
| :--- | :--- | :--- |
| 默认故障复盘报告模板 | 中文 | 综述、根因分析、影响范围、处理时间线、改进措施、经验教训 |
| Default Post Mortem Report Template | 英文 | Summary, Root Cause, Impact, Timeline, Corrective Actions, Lessons Learned |

这两个模板覆盖了故障复盘中最常见的章节，适用于大多数场景。如果你的团队有特殊需求，可以在此基础上创建自定义模板。

<Note>
内置模板不可修改或删除。
</Note>

### 每个章节的作用

| 章节 | 回答什么问题 |
| :--- | :--- |
| **综述** | 发生了什么？用几句话让没有参与处理的人快速了解全貌 |
| **根因分析** | 为什么会发生？直接原因和根本原因分别是什么 |
| **影响范围** | 影响了多少用户、哪些业务、持续了多久 |
| **处理时间线** | 从发现到恢复，每一步做了什么、什么时候做的 |
| **改进措施** | 接下来要做什么来防止同类问题再次发生 |
| **经验教训** | 这次经历让团队学到了什么 |

## 创建自定义模板

如果内置模板不能完全满足你的需求——比如你的团队需要增加“客户沟通记录”或“成本影响评估”章节，可以创建自定义模板。

<Steps>
<Step title="进入模板管理">
在复盘报告页面，点击"模板管理"。
</Step>

<Step title="填写基本信息">

| 字段 | 必填 | 说明 |
| :--- | :---: | :--- |
| 模板名称 | 是 | 起一个一目了然的名字，比如"支付业务线复盘模板" |
| 模板描述 | 否 | 简要说明这个模板适用于什么场景 |
| 所属团队 | 是 | 选择负责维护此模板的团队 |
| 模板内容 | 是 | 在编辑器中定义章节结构 |
</Step>

<Step title="设计章节结构">
在编辑器中定义模板的章节。建议用二级标题作为章节标题，每个章节下加一段引用文本作为填写指引，告诉使用者这个章节应该写什么。
</Step>

<Step title="保存">
点击"保存"完成创建。新模板会立即出现在模板列表中，所有用户都可以在创建复盘报告时选用。
</Step>
</Steps>

<Tip>
模板的章节结构也会作为 AI 生成内容时的参考框架，因此清晰的章节定义不仅帮助人写得更好，也能让 AI 的输出更有针对性。
</Tip>

## 编辑和删除模板

在模板列表中点击自定义模板即可编辑名称、描述、所属团队和内容。不再需要的模板可以删除。

<Tip>
删除模板不会影响已经使用该模板创建的复盘报告。
</Tip>

## 谁可以管理模板

任何人都可以创建模板，但编辑和删除权限受团队归属控制：

| 角色 | 编辑和删除 |
| :--- | :--- |
| 账户级用户 / 账户管理员 | 可以管理所有模板 |
| 模板所属团队的成员 | 可以管理本团队的模板 |
| 其他用户 | 仅可查看和使用，不可编辑或删除 |

<Tip>
虽然编辑删除有权限限制，但所有用户在创建复盘报告时都可以选用任何模板——包括其他团队创建的自定义模板。
</Tip>
