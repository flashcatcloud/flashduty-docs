## 概述

Monitors 支持告警规则的导入导出，方便在不同环境之间迁移规则配置，或对现有规则进行备份。

## 导出规则

**菜单入口**：告警规则 → 导出

选择需要导出的规则或规则分组，导出为 JSON 格式文件。导出内容包含规则的完整配置（查询语句、阈值条件、通知设置等）。

## 导入规则

**菜单入口**：告警规则 → 导入

### 从 JSON 导入

上传之前导出的 JSON 格式文件，系统会解析并创建对应的告警规则。导入时需要指定目标规则分组和关联的数据源。

### 从 Prometheus 告警规则导入

支持导入标准的 Prometheus AlertRules YAML 格式，以 `groups` 为根节点。系统会自动将 Prometheus 规则转换为 Monitors 告警规则。

```yaml
groups:
  - name: example
    rules:
      - alert: HighMemoryUsage
        expr: node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes < 0.1
        for: 5m
        labels:
          severity: critical
```

<Note>
YAML 缩进必须正确，否则会导入失败。导入后建议检查规则配置，确认数据源关联和阈值条件是否符合预期。
</Note>

### 从规则模板导入

可从规则模板库中选择预定义的模板，快速创建告警规则。详见[规则模板库](/zh/monitors/quickstart/rulesets)。
