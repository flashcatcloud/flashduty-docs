## 前置条件

在开始之前，请确保您已准备好以下内容：

- **Terraform >= 1.0** — 如尚未安装，请参考 [Terraform 官方安装指南](https://developer.hashicorp.com/terraform/downloads)。
- **Flashduty APP Key** — 在 Flashduty 控制台的「账户设置 -> APP Key」中获取。此 Key 用于 Provider 调用 Flashduty API。

<Warning>APP Key 拥有账户级别的管理权限，请妥善保管，不要将其硬编码在公开的代码仓库中。建议通过环境变量传入。</Warning>

## 配置 Provider

<Steps>
  <Step title="创建项目目录">
    创建一个新目录用于存放 Terraform 配置文件：

    ```bash
    mkdir flashduty-terraform && cd flashduty-terraform
    ```
  </Step>
  <Step title="创建配置文件">
    创建 `main.tf` 文件，声明 Provider 来源和认证配置：

    <CodeGroup>
    ```hcl main.tf（环境变量方式，推荐）
    terraform {
      required_providers {
        flashduty = {
          source  = "flashcatcloud/flashduty"
          version = "~> 0.1"
        }
      }
    }

    provider "flashduty" {
      # APP Key 通过环境变量 FLASHDUTY_APP_KEY 传入
    }
    ```

    ```hcl main.tf（直接配置方式）
    terraform {
      required_providers {
        flashduty = {
          source  = "flashcatcloud/flashduty"
          version = "~> 0.1"
        }
      }
    }

    provider "flashduty" {
      app_key = "your-app-key"
    }
    ```
    </CodeGroup>

    <Tip>推荐使用环境变量方式传入 APP Key，避免敏感信息泄露。设置方法：`export FLASHDUTY_APP_KEY="your-app-key"`</Tip>
  </Step>
  <Step title="初始化 Provider">
    运行 `terraform init` 下载并安装 Flashduty Provider：

    ```bash
    terraform init
    ```

    成功后您会看到类似以下输出：

    ```
    Initializing provider plugins...
    - Finding flashcatcloud/flashduty versions matching "~> 0.1"...
    - Installing flashcatcloud/flashduty v0.1.x...
    - Installed flashcatcloud/flashduty v0.1.x

    Terraform has been successfully initialized!
    ```
  </Step>
</Steps>

### Provider 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `app_key` | String | 否 | Flashduty APP Key，也可通过 `FLASHDUTY_APP_KEY` 环境变量设置 |
| `base_url` | String | 否 | API 基础 URL，默认为 `https://api.flashcat.cloud`，也可通过 `FLASHDUTY_BASE_URL` 环境变量设置 |

## 第一个示例：创建团队和协作空间

在 `main.tf` 中添加以下资源定义，创建一个团队并在该团队下创建一个协作空间：

```hcl
# 创建一个 SRE 团队
resource "flashduty_team" "sre" {
  team_name   = "SRE Team"
  description = "Site Reliability Engineering team"
}

# 在 SRE 团队下创建一个协作空间
resource "flashduty_channel" "production" {
  channel_name = "Production Alerts"
  description  = "Channel for production environment alerts"
  team_id      = tonumber(flashduty_team.sre.id)
}
```

## 执行部署

<Steps>
  <Step title="预览变更">
    运行 `terraform plan` 预览将要创建的资源：

    ```bash
    terraform plan
    ```

    Terraform 会显示计划创建的资源列表，但不会实际执行任何操作。确认输出内容符合预期后，继续下一步。
  </Step>
  <Step title="应用变更">
    运行 `terraform apply` 创建资源：

    ```bash
    terraform apply
    ```

    Terraform 会再次显示计划，并要求您输入 `yes` 确认执行。输入确认后，资源将被创建。
  </Step>
  <Step title="查看结果">
    运行 `terraform show` 查看当前管理的资源状态：

    ```bash
    terraform show
    ```

    您也可以登录 Flashduty 控制台，确认团队和协作空间已成功创建。
  </Step>
</Steps>

## 更多示例

### 邀请成员并配置分派策略

```hcl
# 邀请新成员
resource "flashduty_member_invite" "john" {
  email       = "john@example.com"
  member_name = "John Doe"
}

# 为协作空间创建分派策略
resource "flashduty_escalate_rule" "default" {
  channel_id  = tonumber(flashduty_channel.production.id)
  rule_name   = "Default Escalation"
  description = "Default escalation policy"
  template_id = "your-template-id"

  layers = [
    {
      escalate_window = 15
      target = {
        person_ids = [flashduty_member_invite.john.member_id]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
      }
    }
  ]
}
```

### 销毁资源

当您不再需要这些资源时，可以运行以下命令销毁：

```bash
terraform destroy
```

<Warning>此操作会删除 Terraform 管理的所有资源，请在执行前仔细确认。</Warning>

## 状态管理

<Note>
Terraform 通过状态文件（`terraform.tfstate`）跟踪已创建资源与配置文件之间的映射关系。

- **不要手动修改** 状态文件。
- **不要将状态文件提交到公开仓库**，因为它可能包含敏感信息。
- 在团队协作中，建议使用 [远程后端](https://developer.hashicorp.com/terraform/language/backend)（如 S3、Consul 等）存储状态文件，避免冲突。
</Note>
