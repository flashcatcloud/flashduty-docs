## 什么是 Terraform

[Terraform](https://www.terraform.io/) 是 HashiCorp 推出的开源基础设施即代码（Infrastructure as Code，简称 IaC）工具。它允许您通过声明式的配置文件来定义和管理基础设施资源，而非手动在控制台中逐一操作。Terraform 支持数千种云服务和 SaaS 平台的 Provider，Flashduty 即是其中之一。

## Flashduty Terraform Provider

Flashduty Terraform Provider 允许您通过 Terraform 配置文件来管理 Flashduty 平台中的各类资源。Provider 已发布至 [Terraform Registry](https://registry.terraform.io/providers/flashcatcloud/flashduty/latest)，您可以直接引用。

### 支持的资源

Provider 目前支持 **12 种资源（Resource）** 的创建和管理：

| 资源 | 说明 |
|------|------|
| `flashduty_team` | 团队管理 |
| `flashduty_member_invite` | 邀请成员加入账户 |
| `flashduty_channel` | 协作空间管理，支持告警聚合和抖动检测 |
| `flashduty_schedule` | 值班表管理，支持轮换规则配置 |
| `flashduty_incident` | 通过代码创建和管理故障 |
| `flashduty_escalate_rule` | 分派策略，支持多环节分层通知 |
| `flashduty_silence_rule` | 静默策略，在维护期间静默告警 |
| `flashduty_inhibit_rule` | 抑制策略，基于条件抑制告警 |
| `flashduty_field` | 自定义字段，为故障添加元数据 |
| `flashduty_route` | 告警路由，配置共享集成的告警分发规则 |
| `flashduty_template` | 通知模板，管理跨渠道的通知模板 |
| `flashduty_alert_pipeline` | 告警处理管道，定义告警的转换、丢弃和抑制规则 |

### 支持的数据源

Provider 还支持 **13 种数据源（Data Source）**，用于查询已有资源：

| 数据源 | 说明 |
|--------|------|
| `flashduty_team` / `flashduty_teams` | 查询团队 |
| `flashduty_channel` / `flashduty_channels` | 查询协作空间 |
| `flashduty_member` / `flashduty_members` | 查询成员 |
| `flashduty_field` / `flashduty_fields` | 查询自定义字段 |
| `flashduty_template` / `flashduty_templates` | 查询通知模板 |
| `flashduty_route` / `flashduty_route_history` | 查询路由规则及变更历史 |
| `flashduty_alert_pipeline` | 查询告警处理管道规则 |

## 何时使用 Terraform

<Tip>如果您的团队已经在使用 Terraform 管理其他基础设施，Flashduty Provider 可以让您将告警管理配置纳入同一套 IaC 工作流中。</Tip>

**适合使用 Terraform 的场景：**

- **多环境一致性** — 您需要在开发、测试、生产等多个环境中维护相同的告警配置，Terraform 可以确保配置一致。
- **版本控制和审计** — 所有配置变更通过 Git 管理，每次变更都有记录，支持代码审查和回滚。
- **批量管理** — 需要同时管理大量团队、协作空间或分派策略时，代码化管理远比手动操作高效。
- **自动化流水线** — 将 Flashduty 配置集成到 CI/CD 流水线中，实现基础设施变更的自动化部署。
- **跨平台编排** — 将 Flashduty 资源与云资源、监控系统等统一编排，在一套配置中管理整个基础设施栈。

**适合使用控制台 UI 的场景：**

- 需要快速创建或修改单个资源。
- 需要实时查看故障状态和处理进度。
- 团队成员不熟悉 Terraform 或 IaC 工具。
