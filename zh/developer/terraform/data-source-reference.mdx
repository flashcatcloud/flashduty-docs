本页列出 Flashduty Terraform Provider 支持的全部 13 种数据源。数据源用于查询已有的 Flashduty 资源，以便在 Terraform 配置中引用。

<Tip>数据源以 `data` 块声明，通过 `data.<类型>.<名称>.<属性>` 引用输出值。单数数据源按 ID 查询单个资源，复数数据源查询资源列表。</Tip>

<Accordion title="flashduty_team — 查询单个团队" icon="user-group">

按 ID、名称或外部引用 ID 查询单个团队。

### 查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `team_id` | Number | 团队 ID，三个参数至少填一个 |
| `team_name` | String | 团队名称 |
| `ref_id` | String | 外部引用 ID |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `team_name` | String | 团队名称 |
| `description` | String | 团队描述 |
| `created_at` | Number | 创建时间戳 |
| `updated_at` | Number | 更新时间戳 |

### 示例

```hcl
# 按 ID 查询
data "flashduty_team" "by_id" {
  team_id = 12345
}

# 按名称查询
data "flashduty_team" "by_name" {
  team_name = "SRE Team"
}

# 按外部引用 ID 查询
data "flashduty_team" "by_ref" {
  ref_id = "sre-team-ref"
}

output "team_name" {
  value = data.flashduty_team.by_id.team_name
}
```

</Accordion>

<Accordion title="flashduty_teams — 查询团队列表" icon="users">

查询多个团队，支持按名称或描述搜索。

### 查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `query` | String | 按名称或描述搜索团队 |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `teams` | List of Object | 团队列表，每个包含 `team_id`、`team_name`、`description` |

### 示例

```hcl
# 查询所有团队
data "flashduty_teams" "all" {}

output "all_team_names" {
  value = [for team in data.flashduty_teams.all.teams : team.team_name]
}

# 按名称搜索
data "flashduty_teams" "sre" {
  query = "SRE"
}

# 通过 local 查找特定团队
locals {
  sre_team = [for team in data.flashduty_teams.all.teams : team if team.team_name == "SRE Team"][0]
}

output "sre_team_id" {
  value = local.sre_team.team_id
}
```

</Accordion>

<Accordion title="flashduty_channel — 查询单个协作空间" icon="layer-group">

按 ID 查询单个协作空间的详细信息。

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `channel_id` | Number | 是 | 协作空间 ID |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_name` | String | 协作空间名称 |
| `description` | String | 协作空间描述 |
| `team_id` | Number | 所属团队 ID |
| `status` | String | 状态 |
| `is_private` | Boolean | 是否为私有空间 |
| `auto_resolve_mode` | String | 自动恢复模式 |
| `auto_resolve_timeout` | Number | 自动恢复超时（秒） |
| `disable_auto_close` | Boolean | 是否禁用自动关闭 |
| `disable_outlier_detection` | Boolean | 是否禁用新奇故障检测 |
| `managing_team_ids` | List of Number | 管理团队 ID 列表 |
| `created_at` | Number | 创建时间戳 |
| `updated_at` | Number | 更新时间戳 |

### 示例

```hcl
data "flashduty_channel" "production" {
  channel_id = 6148622168454
}

output "channel_name" {
  value = data.flashduty_channel.production.channel_name
}

output "channel_team_id" {
  value = data.flashduty_channel.production.team_id
}
```

</Accordion>

<Accordion title="flashduty_channels — 查询协作空间列表" icon="layer-group">

查询多个协作空间，支持按名称搜索和按团队过滤。

### 查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `query` | String | 按名称或描述搜索 |
| `team_ids` | List of Number | 按团队 ID 过滤 |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `channels` | List of Object | 协作空间列表，每个包含 `channel_id`、`channel_name`、`description`、`status`、`team_id` |

### 示例

```hcl
# 查询所有协作空间
data "flashduty_channels" "all" {}

output "all_channel_names" {
  value = [for ch in data.flashduty_channels.all.channels : ch.channel_name]
}

# 按团队过滤
data "flashduty_channels" "sre_channels" {
  team_ids = [12345]
}

# 按名称搜索
data "flashduty_channels" "production" {
  query = "production"
}
```

</Accordion>

<Accordion title="flashduty_members — 查询成员列表" icon="users">

查询多个成员，支持按名称、邮箱或手机号搜索。

### 查询参数

| 参数 | 类型 | 说明 |
|------|------|------|
| `query` | String | 按名称、邮箱或手机号搜索 |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `members` | List of Object | 成员列表，每个包含 `member_id`、`member_name`、`email`、`status` |

### 示例

```hcl
# 查询所有成员
data "flashduty_members" "all" {}

output "all_member_names" {
  value = [for m in data.flashduty_members.all.members : m.member_name]
}

# 搜索特定成员
data "flashduty_members" "search" {
  query = "john@example.com"
}

locals {

}

output "active_member_count" {
  value = length(local.active_members)
}
```

<Accordion title="flashduty_field — 查询单个自定义字段" icon="tag">

按 ID 查询单个自定义字段的详细信息。

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | String | 是 | 字段唯一标识 |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `field_name` | String | 字段名称 |
| `display_name` | String | 显示名称 |
| `description` | String | 字段描述 |
| `field_type` | String | 字段类型 |
| `value_type` | String | 值类型 |
| `default_value` | String | 默认值（JSON） |
| `options` | String | 可选项（JSON 数组） |
| `status` | String | 字段状态 |
| `created_at` | Number | 创建时间戳 |
| `updated_at` | Number | 更新时间戳 |

### 示例

```hcl
data "flashduty_field" "priority" {
  id = "field-abc123"
}

output "field_name" {
  value = data.flashduty_field.priority.field_name
}

output "field_display_name" {
  value = data.flashduty_field.priority.display_name
}

output "field_options" {
  value = data.flashduty_field.priority.options != null ? jsondecode(data.flashduty_field.priority.options) : []
}
```

</Accordion>

<Accordion title="flashduty_fields — 查询自定义字段列表" icon="tags">

查询所有自定义字段。

### 查询参数

无。

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `fields` | List of Object | 字段列表，每个包含 `id`、`field_name`、`display_name`、`description`、`field_type`、`value_type`、`default_value`、`options`、`status`、`created_at`、`updated_at` |

### 示例

```hcl
data "flashduty_fields" "all" {}

output "all_field_names" {
  value = [for f in data.flashduty_fields.all.fields : f.display_name]
}

# 查找选择类型字段
locals {
  select_fields = [for f in data.flashduty_fields.all.fields : f if f.field_type == "single_select" || f.field_type == "multi_select"]
}

output "select_field_count" {
  value = length(local.select_fields)
}
```

</Accordion>

<Accordion title="flashduty_route — 查询告警路由" icon="route">

按集成 ID 查询当前生效的告警路由配置。

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `integration_id` | Number | 是 | 共享集成 ID |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 路由唯一标识 |
| `status` | String | 路由状态 |
| `version` | Number | 路由配置版本号 |
| `cases` | List of Object | 条件路由规则，每个包含 `if`（过滤条件）、`channel_ids`、`routing_mode`、`name_mapping_label`、`fallthrough` |
| `default` | Object | 默认路由配置，含 `channel_ids` |
| `sections` | List of Object | 路由规则分组，含 `name`、`position` |
| `created_at` | Number | 创建时间戳 |
| `updated_at` | Number | 更新时间戳 |

### 示例

```hcl
data "flashduty_route" "main" {
  integration_id = 5592304204454
}

output "route_version" {
  value = data.flashduty_route.main.version
}

output "route_status" {
  value = data.flashduty_route.main.status
}

output "default_channels" {
  value = data.flashduty_route.main.default != null ? data.flashduty_route.main.default.channel_ids : []
}

output "route_cases_count" {
  value = data.flashduty_route.main.cases != null ? length(data.flashduty_route.main.cases) : 0
}
```

</Accordion>

<Accordion title="flashduty_route_history — 查询路由变更历史" icon="clock-rotate-left">

查询共享集成的告警路由变更历史记录。

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `integration_id` | Number | 是 | 共享集成 ID |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `items` | List of Object | 历史记录列表，每个包含以下字段 |

每条历史记录包含：

| 属性 | 类型 | 说明 |
|------|------|------|
| `version` | Number | 版本号 |
| `updated_at` | Number | 更新时间戳 |
| `updated_by` | Number | 更新者用户 ID |
| `cases` | List of Object | 该版本的条件路由规则 |
| `default` | Object | 该版本的默认路由配置 |
| `sections` | List of Object | 该版本的规则分组 |

### 示例

```hcl
data "flashduty_route_history" "main" {
  integration_id = 5592304204454
}

output "history_count" {
  value = length(data.flashduty_route_history.main.items)
}

output "latest_version" {
  value = length(data.flashduty_route_history.main.items) > 0 ? data.flashduty_route_history.main.items[0].version : 0
}

output "all_versions" {
  value = [for item in data.flashduty_route_history.main.items : item.version]
}
```

</Accordion>

<Accordion title="flashduty_alert_pipeline — 查询告警处理管道" icon="filter">

按集成 ID 查询告警处理管道规则。

### 查询参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `integration_id` | Number | 是 | 集成 ID |

### 输出属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `rules` | List of Object | 管道规则列表，每个包含以下字段 |

每条规则包含：

| 属性 | 类型 | 说明 |
|------|------|------|
| `kind` | String | 规则类型：`title_reset`、`description_reset`、`severity_reset`、`alert_drop`、`alert_inhibit` |
| `settings` | String | 规则设置（JSON 格式） |
| `if` | List of Object | 过滤条件，每个含 `key`、`oper`、`vals` |

### 示例

```hcl
data "flashduty_alert_pipeline" "example" {
  integration_id = 5592304204454
}

output "pipeline_rules" {
  value = data.flashduty_alert_pipeline.example.rules
}

output "rule_count" {
  value = length(data.flashduty_alert_pipeline.example.rules)
}
```

</Accordion>
