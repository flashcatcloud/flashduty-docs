本页列出 Flashduty Terraform Provider 支持的全部 12 种资源。每种资源包含完整的属性说明和 HCL 配置示例。

<Tip>所有资源名称以 `flashduty_` 为前缀。创建资源后，Terraform 自动生成的 `id` 属性可用于其他资源的引用。</Tip>

<Accordion title="flashduty_team — 团队" icon="users">

管理 Flashduty 团队。团队是组织成员和资源的基本单位。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `team_name` | String | 团队名称，在账户内必须唯一 |
| `description` | String | 团队描述 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 团队唯一标识 |

### 示例

```hcl
resource "flashduty_team" "sre" {
  team_name   = "SRE Team"
  description = "Site Reliability Engineering team"
}

resource "flashduty_team" "payment" {
  team_name   = "Payment Service Team"
  description = "Team responsible for payment service operations"
}
```

</Accordion>

<Accordion title="flashduty_member_invite — 邀请成员" icon="user-plus">

邀请成员加入 Flashduty 账户。被邀请的成员创建后可登录激活账户。

### 属性

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `email` | String | 成员邮箱，`email` 和 `phone` 至少填一项 |
| `phone` | String | 成员手机号，`email` 和 `phone` 至少填一项 |
| `member_name` | String | 成员名称，使用手机号邀请时必填 |
| `country_code` | String | 手机号国家码，默认 `CN` |
| `ref_id` | String | 外部引用 ID |
| `role_ids` | List of Number | 分配给成员的角色 ID 列表 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 邀请唯一标识 |
| `member_id` | Number | 邀请后分配的 Flashduty 成员 ID |

### 示例

```hcl
# 通过邮箱和手机号邀请
resource "flashduty_member_invite" "john" {
  email        = "john.doe@example.com"
  member_name  = "John Doe"
  country_code = "CN"
  phone        = "10000000000"
}

# 仅通过邮箱邀请
resource "flashduty_member_invite" "alice" {
  email       = "alice@example.com"
  member_name = "Alice Smith"
}
```

</Accordion>

<Accordion title="flashduty_channel — 协作空间" icon="layer-group">

管理 Flashduty 协作空间，支持告警聚合和抖动检测配置。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_name` | String | 协作空间名称 |
| `team_id` | Number | 所属团队 ID |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `description` | String | 协作空间描述 |
| `is_private` | Boolean | 是否为私有空间，默认 `false` |
| `auto_resolve_timeout` | Number | 自动恢复超时时间（秒），范围 0-86400，不设置则禁用 |
| `auto_resolve_mode` | String | 自动恢复模式：`trigger`（触发后计时）、`update`（更新后计时） |
| `disable_auto_close` | Boolean | 是否禁用自动关闭故障，默认 `false` |
| `disable_outlier_detection` | Boolean | 是否禁用新奇故障检测，默认 `false` |
| `managing_team_ids` | List of Number | 管理团队 ID 列表（最多 3 个） |
| `group` | Object | 告警聚合配置，详见下方 |
| `flapping` | Object | 抖动检测配置，详见下方 |

**`group` 告警聚合配置：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `method` | String | **必填**。聚合方式：`i`（智能告警聚合）、`p`（规则告警聚合）、`n`（不聚合） |
| `time_window` | Number | 聚合时间窗口（分钟），范围 0-60，0 表示合并至故障关闭 |
| `i_keys` | List of String | 智能聚合参考字段，如 `title`、`description`、`labels.service` |
| `i_score_threshold` | Number | 智能聚合相似度阈值，范围 0.5-1.0，默认 0.9 |
| `equals` | List of List of String | 规则聚合默认维度（OR 组合中的 AND 组） |
| `cases` | List of Object | 规则聚合分支规则（最多 10 条），自上而下匹配 |
| `storm_thresholds` | List of Number | 告警风暴预警阈值（最多 5 个值，每个 2-10000） |
| `all_equals_required` | Boolean | 是否要求所有聚合维度都存在，默认 `false` |

**`flapping` 抖动检测配置：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `is_disabled` | Boolean | **必填**。是否禁用抖动检测 |
| `max_changes` | Number | 时间窗口内最大状态变化次数，范围 2-100，默认 4 |
| `in_mins` | Number | 统计时间窗口（分钟），范围 1-1440，默认 60 |
| `mute_mins` | Number | 静默时间窗口（分钟），范围 0-1440，默认 120，0 表示不静默 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 协作空间唯一标识 |

### 示例

```hcl
resource "flashduty_team" "sre" {
  team_name   = "SRE Team"
  description = "Site Reliability Engineering team"
}

# 基础协作空间
resource "flashduty_channel" "basic" {
  channel_name = "Production Alerts"
  description  = "Channel for production environment alerts"
  team_id      = tonumber(flashduty_team.sre.id)
}

# 智能告警聚合
resource "flashduty_channel" "intelligent" {
  channel_name = "API Service Alerts"
  description  = "Alerts with intelligent grouping"
  team_id      = tonumber(flashduty_team.sre.id)

  group = {
    method            = "i"
    time_window       = 10
    i_score_threshold = 0.85
    i_keys            = ["title", "description", "labels.service"]
    storm_thresholds  = [100, 500]
  }
}

# 规则告警聚合 + 抖动检测
resource "flashduty_channel" "rule_based" {
  channel_name = "Database Alerts"
  description  = "Database alerts with rule-based grouping"
  team_id      = tonumber(flashduty_team.sre.id)

  group = {
    method      = "p"
    time_window = 30
    equals      = [["title", "labels.host"]]

    cases = [
      {
        if = [
          {
            key  = "severity"
            oper = "IN"
            vals = ["Critical"]
          }
        ]
        equals = ["title", "labels.host", "labels.database"]
      }
    ]
  }

  flapping = {
    is_disabled = false
    max_changes = 5
    in_mins     = 60
    mute_mins   = 120
  }
}
```

</Accordion>

<Accordion title="flashduty_schedule — 值班表" icon="calendar">

管理 Flashduty 值班表，支持轮换规则、时间限制和通知配置。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `schedule_name` | String | 值班表名称 |
| `description` | String | 值班表描述 |
| `layers` | List of Object | 值班层级（轮换规则），详见下方 |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `team_id` | Number | 所属团队 ID |
| `status` | Number | 值班表状态，0 = 启用 |
| `notify` | Object | 通知设置，详见下方 |

**`layers` 值班层级配置：**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `layer_name` | String | 是 | 层级名称 |
| `mode` | Number | 是 | 模式：0 = 常规，1 = 临时 |
| `layer_start` | Number | 是 | 层级开始时间（Unix 时间戳） |
| `rotation_unit` | String | 是 | 轮换单位：`hour`、`day`、`week`、`month` |
| `rotation_value` | Number | 是 | 轮换间隔值 |
| `groups` | List of Object | 是 | 值班分组，包含 `group_name`（可选）和 `members`（必填，含 `person_ids` 和可选 `role_id`） |
| `layer_end` | Number | 否 | 层级结束时间（Unix 时间戳） |
| `fair_rotation` | Boolean | 否 | 是否启用公平轮换 |
| `handoff_time` | Number | 否 | 交接时间（秒） |
| `restrict_mode` | Number | 否 | 限制模式，0 = 无限制 |
| `restrict_periods` | List of Object | 否 | 时间限制区间，含 `restrict_start` 和 `restrict_end`（从午夜起的秒数） |
| `day_mask` | Object | 否 | 工作日掩码，`repeat` 为星期列表（0=周日，1=周一，...，6=周六） |
| `mask_continuous_enabled` | Boolean | 否 | 是否启用连续掩码 |

**`notify` 通知配置：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `advance_in_time` | Number | 提前通知时间（秒） |
| `by` | Object | 通知渠道，含 `follow_preference`（Boolean）和 `personal_channels`（如 `email`、`sms`、`phone`） |
| `fixed_time` | Object | 固定时间通知，含 `cycle` 和 `start` |
| `webhooks` | List of Object | Webhook 通知，含 `type` 和 `settings`（JSON 字符串） |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 值班表唯一标识 |

### 示例

```hcl
# 简单的每日轮换值班表
resource "flashduty_schedule" "primary" {
  schedule_name = "Primary On-Call"
  description   = "Primary on-call rotation for the SRE team"

  layers = [
    {
      layer_name     = "Primary Layer"
      mode           = 0
      layer_start    = 1704038400
      rotation_unit  = "day"
      rotation_value = 1
      groups = [
        {
          group_name = "SRE Group"
          members = [
            {
              role_id    = 0
              person_ids = [75340551232454]
            }
          ]
        }
      ]
    }
  ]
}

# 带工作时间限制和通知的值班表
resource "flashduty_schedule" "business_hours" {
  schedule_name = "Business Hours Schedule"
  description   = "Weekday business hours on-call"

  layers = [
    {
      layer_name     = "Business Hours"
      mode           = 0
      layer_start    = 1704038400
      rotation_unit  = "week"
      rotation_value = 1
      fair_rotation  = true
      restrict_mode  = 1
      restrict_periods = [
        {
          restrict_start = 32400
          restrict_end   = 64800
        }
      ]
      day_mask = {
        repeat = [1, 2, 3, 4, 5]
      }
      groups = [
        {
          group_name = "Weekday Team"
          members = [
            {
              role_id    = 0
              person_ids = [75340551232454]
            }
          ]
        }
      ]
    }
  ]

  notify = {
    advance_in_time = 3600
    by = {
      follow_preference = true
      personal_channels = ["email"]
    }
  }
}
```

</Accordion>

<Accordion title="flashduty_incident — 故障" icon="triangle-exclamation">

通过代码创建和管理 Flashduty 故障。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `title` | String | 故障标题 |
| `incident_severity` | String | 故障级别：`Critical`（严重）、`Warning`（一般）、`Info`（轻微） |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_id` | Number | 所属协作空间 ID |
| `description` | String | 故障描述，支持纯文本或 Markdown |
| `impact` | String | 影响描述 |
| `root_cause` | String | 根因分析 |
| `resolution` | String | 解决方案 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 故障唯一标识（ObjectID） |
| `incident_status` | String | 当前状态：`Info`、`Warning`、`Critical`、`Ok` |
| `progress` | String | 处理进度：`Triggered`（待处理）、`Processing`（处理中）、`Closed`（已关闭） |

### 示例

```hcl
resource "flashduty_incident" "database_outage" {
  title             = "Database Connection Failure"
  description       = "Primary database is not responding to connection requests"
  incident_severity = "Critical"
  channel_id        = tonumber(flashduty_channel.production.id)
}

resource "flashduty_incident" "high_latency" {
  title             = "API Latency Spike"
  description       = "API response times exceeding SLA thresholds"
  incident_severity = "Warning"
  channel_id        = tonumber(flashduty_channel.production.id)
}
```

</Accordion>

<Accordion title="flashduty_escalate_rule — 分派策略" icon="arrow-up-right-dots">

管理协作空间的分派策略，支持多环节分层通知。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_id` | Number | 所属协作空间 ID |
| `rule_name` | String | 策略名称 |
| `template_id` | String | 通知模板 ID |
| `layers` | List of Object | 分派环节列表，详见下方 |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `description` | String | 策略描述 |
| `aggr_window` | Number | 聚合窗口（秒），范围 0-3600 |
| `priority` | Number | 策略优先级 |
| `filters` | List of Object | 告警匹配过滤条件（组间 OR，条件间 AND） |
| `time_filters` | List of Object | 时间过滤条件，控制策略生效时段 |

**`layers` 分派环节配置：**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `target` | Object | 是 | 分派目标，详见下方 |
| `escalate_window` | Number | 否 | 升级等待时间（分钟），默认 30 |
| `max_times` | Number | 否 | 最大通知次数，默认 2 |
| `notify_step` | Number | 否 | 通知间隔（分钟），默认 10 |
| `force_escalate` | Boolean | 否 | 是否在已认领后仍升级 |

**`target` 分派目标配置：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `person_ids` | List of Number | 处理人员 ID 列表 |
| `team_ids` | List of Number | 团队 ID 列表 |
| `schedule_ids` | List of Number | 值班表 ID 列表 |
| `emails` | List of String | 邮箱地址列表 |
| `by` | Object | 通知方式配置，含 `follow_preference`（Boolean）和按级别的通知渠道 `critical`、`warning`、`info`（如 `email`、`sms`、`voice`、`im`） |
| `webhooks` | List of Object | Webhook 通知目标，含 `type`（如 `feishu_app`、`wecom`、`dingtalk`、`slack`）和 `settings`（JSON 字符串） |

**`filters` 过滤条件：**

每个 filter 包含 `conditions` 列表（AND 关系），filter 之间为 OR 关系。每个 condition 包含 `key`（如 `severity`、`labels.xxx`）、`oper`（`IN` 或 `NOTIN`）、`vals`（匹配值列表）。

**`time_filters` 时间过滤条件：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `start` | String | 开始时间（HH:MM 格式） |
| `end` | String | 结束时间（HH:MM 格式） |
| `repeat` | List of Number | 生效星期（0=周日，1=周一，...，6=周六） |
| `cal_id` | String | 日历 ID |
| `is_off` | Boolean | 是否匹配日历的休息时间 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 分派策略唯一标识 |

### 示例

```hcl
# 基础分派策略
resource "flashduty_escalate_rule" "default" {
  channel_id  = 6148622168454
  rule_name   = "Default Escalation"
  description = "Default escalation policy for all alerts"
  template_id = "6321aad26c12104586a88916"

  layers = [
    {
      escalate_window = 15
      target = {
        person_ids = [75340551232454]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
      }
    },
    {
      escalate_window = 30
      target = {
        schedule_ids = [4973305527804]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
      }
    }
  ]
}

# 带过滤条件和时间限制的分派策略
resource "flashduty_escalate_rule" "critical" {
  channel_id  = 6148622168454
  rule_name   = "Critical Alerts Escalation"
  description = "Escalation for critical alerts during business hours"
  template_id = "6321aad26c12104586a88916"
  aggr_window = 60

  layers = [
    {
      max_times       = 3
      notify_step     = 10
      escalate_window = 15
      force_escalate  = true
      target = {
        person_ids = [75340551232454]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
        webhooks = [
          {
            type     = "wecom"
            settings = jsonencode({ token = "https://qyapi.weixin.qq.com/xxx" })
          }
        ]
      }
    }
  ]

  time_filters = [
    {
      start  = "09:00"
      end    = "18:00"
      repeat = [1, 2, 3, 4, 5]
    }
  ]

  filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Critical"]
        }
      ]
    }
  ]
}
```

</Accordion>

<Accordion title="flashduty_silence_rule — 静默策略" icon="volume-xmark">

管理协作空间的静默策略，在维护窗口期间静默告警通知。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_id` | Number | 所属协作空间 ID |
| `rule_name` | String | 策略名称 |
| `description` | String | 策略描述 |
| `filters` | List of Object | 过滤条件（组间 OR，条件间 AND），每个 condition 含 `key`、`oper`（`IN`/`NOTIN`）、`vals` |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `is_directly_discard` | Boolean | 是否直接丢弃匹配的告警 |
| `priority` | Number | 策略优先级 |
| `time_filter` | Object | 单次时间范围，含 `start_time` 和 `end_time`（Unix 时间戳），与 `time_filters` 互斥 |
| `time_filters` | List of Object | 周期性时间条件，含 `start`（HH:MM）、`end`（HH:MM）、`repeat`（星期列表），与 `time_filter` 互斥 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 静默策略唯一标识 |

### 示例

```hcl
# 固定时间窗口的静默策略
resource "flashduty_silence_rule" "maintenance" {
  channel_id  = 6148622168454
  rule_name   = "Weekly Maintenance Window"
  description = "Silence alerts during weekly maintenance"

  filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Warning", "Info"]
        }
      ]
    }
  ]

  time_filter = {
    start_time = 1773121380
    end_time   = 1773207780
  }
}

# 周期性静默策略
resource "flashduty_silence_rule" "known_issue" {
  channel_id  = 6148622168454
  rule_name   = "Known Disk Space Alert"
  description = "Silence known disk space alerts from legacy server"

  filters = [
    {
      conditions = [
        {
          key  = "labels.host"
          oper = "IN"
          vals = ["legacy-server-01"]
        },
        {
          key  = "title"
          oper = "IN"
          vals = ["disk space"]
        }
      ]
    }
  ]

  time_filters = [
    {
      start  = "02:00"
      end    = "04:00"
      repeat = [0]
    }
  ]

  is_directly_discard = true
}
```

</Accordion>

<Accordion title="flashduty_inhibit_rule — 抑制策略" icon="ban">

管理协作空间的抑制策略，基于源告警条件抑制目标告警。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `channel_id` | Number | 所属协作空间 ID |
| `rule_name` | String | 策略名称 |
| `description` | String | 策略描述 |
| `source_filters` | List of Object | 源事件过滤条件（组间 OR，条件间 AND） |
| `target_filters` | List of Object | 目标事件过滤条件（被抑制的事件） |
| `equals` | List of String | 源和目标必须相等的字段（如 `labels.host`） |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `is_directly_discard` | Boolean | 是否直接丢弃匹配的告警 |
| `priority` | Number | 策略优先级 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 抑制策略唯一标识 |

### 示例

```hcl
# Critical 级别抑制 Warning 和 Info 级别
resource "flashduty_inhibit_rule" "critical_inhibits_warning" {
  channel_id  = 6148622168454
  rule_name   = "Critical Inhibits Warning"
  description = "Suppress warning alerts when critical alert is active for same host"

  source_filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Critical"]
        }
      ]
    }
  ]

  target_filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Warning", "Info"]
        }
      ]
    }
  ]

  equals = ["host", "service"]

  is_directly_discard = true
}

# 上游服务故障时抑制下游告警
resource "flashduty_inhibit_rule" "upstream_inhibits_downstream" {
  channel_id  = 6148622168454
  rule_name   = "Upstream Inhibits Downstream"
  description = "Suppress downstream alerts when upstream service is down"

  source_filters = [
    {
      conditions = [
        {
          key  = "labels.service"
          oper = "IN"
          vals = ["database", "cache"]
        }
      ]
    }
  ]

  target_filters = [
    {
      conditions = [
        {
          key  = "labels.service"
          oper = "IN"
          vals = ["api", "web"]
        }
      ]
    }
  ]

  equals = ["datacenter"]

  is_directly_discard = true
}
```

<Accordion title="flashduty_route — 告警路由" icon="route">

管理共享集成的告警路由配置。每个集成只有一个路由配置（upsert 语义），因此每个 `integration_id` 仅定义一个 `flashduty_route` 资源。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `integration_id` | Number | 共享集成 ID |

**可选：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `cases` | List of Object | 条件路由规则，详见下方 |
| `default` | Object | 默认路由配置，含 `channel_ids`（目标协作空间 ID 列表） |
| `sections` | List of Object | 路由规则分组，含 `name`（名称，需唯一）和 `position`（位置，0 表示在第一条规则之前） |

**`cases` 路由规则配置：**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `if` | List of Object | 是 | 过滤条件（AND 关系），每个含 `key`、`oper`（`IN`/`NOTIN`）、`vals` |
| `channel_ids` | List of Number | 否 | 目标协作空间 ID（`standard` 模式时必填） |
| `routing_mode` | String | 否 | 路由模式：`standard`（标准）、`name_mapping`（名称映射） |
| `name_mapping_label` | String | 否 | 名称映射模式下的标签键（如 `labels.ChannelName`） |
| `fallthrough` | Boolean | 否 | 匹配后是否继续匹配下一条规则 |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 路由唯一标识（与 integration_id 相同） |
| `status` | String | 路由状态 |
| `version` | Number | 路由配置版本号 |
| `created_at` | Number | 创建时间戳 |
| `updated_at` | Number | 更新时间戳 |

### 示例

```hcl
resource "flashduty_route" "main" {
  integration_id = 5592304204454

  cases = [
    {
      if = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Critical"]
        }
      ]
      channel_ids  = [6148622168454]
      routing_mode = "standard"
    },
    {
      if = [
        {
          key  = "labels.environment"
          oper = "IN"
          vals = ["production", "prod"]
        },
        {
          key  = "labels.team"
          oper = "IN"
          vals = ["sre", "platform"]
        }
      ]
      channel_ids  = [6148622168454]
      routing_mode = "standard"
    }
  ]

  default = {
    channel_ids = [6148622168454]
  }
}
```

<Accordion title="flashduty_alert_pipeline — 告警处理管道" icon="filter">

管理集成的告警处理管道规则。管道允许您在告警创建故障之前对其进行转换、丢弃或抑制。

### 属性

**必填：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `integration_id` | Number | 所属集成 ID |
| `rules` | List of Object | 管道规则列表（按顺序从上到下执行），详见下方 |

**`rules` 管道规则配置：**

| 属性 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `kind` | String | 是 | 规则类型：`title_reset`（重置标题）、`description_reset`（重置描述）、`severity_reset`（重置级别）、`alert_drop`（丢弃告警）、`alert_inhibit`（抑制告警） |
| `settings` | String | 是 | 规则设置（JSON 格式，使用 `jsonencode()`），内容因 kind 而异 |
| `if` | List of Object | 否 | 过滤条件（AND 关系），每个含 `key`、`oper`（`IN`/`NOTIN`）、`vals` |

**`settings` 配置内容（按 kind）：**

| kind | settings 内容 |
|------|---------------|
| `title_reset` | `{"title": "新标题"}` |
| `description_reset` | `{"description": "新描述"}` |
| `severity_reset` | `{"severity": "Critical"\|"Warning"\|"Info"}` |
| `alert_drop` | `{}` |
| `alert_inhibit` | `{"source_filters": [...], "equals": [...]}` |

**只读：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 资源标识（与 integration_id 相同） |

### 示例

```hcl
resource "flashduty_alert_pipeline" "example" {
  integration_id = 5592304204454

  rules = [
    # 为 Prometheus 来源的告警重置标题
    {
      kind = "title_reset"
      if = [
        {
          key  = "labels.source"
          oper = "IN"
          vals = ["prometheus"]
        }
      ]
      settings = jsonencode({
        title = "[Prometheus] $${title}"
      })
    },

    # 将生产环境 Info 级别告警提升为 Warning
    {
      kind = "severity_reset"
      if = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Info"]
        },
        {
          key  = "labels.env"
          oper = "IN"
          vals = ["production"]
        }
      ]
      settings = jsonencode({
        severity = "Warning"
      })
    },

    # 丢弃已知的测试告警
    {
      kind = "alert_drop"
      if = [
        {
          key  = "title"
          oper = "IN"
          vals = ["test alert", "debug alert"]
        }
      ]
      settings = jsonencode({})
    },

    # 同主机存在 Critical 告警时抑制低级别告警
    {
      kind = "alert_inhibit"
      settings = jsonencode({
        source_filters = [
          {
            key  = "severity"
            oper = "IN"
            vals = ["Critical"]
          }
        ]
        equals = ["labels.host", "labels.service"]
      })
    }
  ]
}
```

</Accordion>
