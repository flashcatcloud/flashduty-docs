## 前置条件

- 一个 Flashduty 账户
- 一个 Flashduty APP Key（在 **账户设置 > APP Key** 中创建）
- 支持 MCP 协议的 AI 工具（如 Claude Desktop、Cursor、Windsurf 等）

<Warning>
APP Key 拥有您账户的完整 API 访问权限，请妥善保管，不要泄露给他人或提交到代码仓库中。
</Warning>

## 选择部署方式

<Tabs>
  <Tab title="远程服务">
    远程服务是最简单的接入方式，无需安装任何组件，直接连接 Flashduty 托管的 MCP 服务。

    **适用场景**：MCP 客户端支持远程 MCP 服务器连接（Streamable HTTP 传输）。

    <Steps>
      <Step title="获取 APP Key">
        登录 Flashduty 控制台，进入 **账户设置 > APP Key**，创建一个新的 APP Key。
      </Step>
      <Step title="配置 MCP 客户端">
        在您的 MCP 客户端中添加以下配置：

        <CodeGroup>
        ```json Cursor
        {
          "mcpServers": {
            "flashduty": {
              "url": "https://mcp.flashcat.cloud/mcp",
              "headers": {
                "Authorization": "Bearer <your_flashduty_app_key>"
              }
            }
          }
        }
        ```
        </CodeGroup>

        将 `<your_flashduty_app_key>` 替换为您的实际 APP Key。
      </Step>
      <Step title="验证连接">
        在 AI 工具中发送一条指令，验证 MCP Server 是否正常工作：

        ```
        请查询当前所有待处理的故障
        ```

        如果配置正确，AI 助手会调用 `query_incidents` 工具并返回故障列表。
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker">
    通过 Docker 在本地运行 MCP Server，适合需要自行托管或客户端不支持远程连接的场景。

    **前置条件**：已安装并启动 [Docker](https://www.docker.com/)。

    <Steps>
      <Step title="获取 APP Key">
        登录 Flashduty 控制台，进入 **账户设置 > APP Key**，创建一个新的 APP Key。
      </Step>
      <Step title="配置 MCP 客户端">
        在您的 MCP 客户端中添加以下配置：

        ```json
        {
          "mcpServers": {
            "flashduty": {
              "command": "docker",
              "args": [
                "run",
                "-i",
                "--rm",
                "-e",
                "FLASHDUTY_APP_KEY",
                "registry.flashcat.cloud/public/flashduty-mcp-server"
              ],
              "env": {
                "FLASHDUTY_APP_KEY": "your_flashduty_app_key"
              }
            }
          }
        }
        ```

        以上配置适用于 Cursor、Claude Desktop 等所有支持 stdio 传输的 MCP 客户端。
      </Step>
      <Step title="验证连接">
        在 AI 工具中发送一条指令测试：

        ```
        请列出所有团队成员
        ```

        如果配置正确，AI 助手会调用 `query_members` 工具并返回成员列表。
      </Step>
    </Steps>

    <Tip>
    您可以通过环境变量进一步配置 Docker 容器，例如启用只读模式或指定工具集。详见 [配置说明](/zh/developer/mcp-server/configuration)。
    </Tip>
  </Tab>

  <Tab title="本地二进制">
    下载预编译的二进制文件或从源码构建，直接在本地运行 MCP Server。

    <Steps>
      <Step title="下载二进制文件">
        从 [GitHub Releases](https://github.com/flashcatcloud/flashduty-mcp-server/releases) 下载适合您操作系统的预编译版本。

        如果您更倾向于从源码构建：

        ```bash
        git clone https://github.com/flashcatcloud/flashduty-mcp-server.git
        cd flashduty-mcp-server
        go build -o flashduty-mcp-server ./cmd/flashduty-mcp-server
        ```
      </Step>
      <Step title="获取 APP Key">
        登录 Flashduty 控制台，进入 **账户设置 > APP Key**，创建一个新的 APP Key。
      </Step>
      <Step title="配置 MCP 客户端">
        您可以通过环境变量或命令行参数提供 APP Key：

        <CodeGroup>
        ```json 环境变量方式
        {
          "mcpServers": {
            "flashduty": {
              "command": "/path/to/flashduty-mcp-server",
              "args": ["stdio"],
              "env": {
                "FLASHDUTY_APP_KEY": "your_app_key_here"
              }
            }
          }
        }
        ```

        ```json 命令行参数方式
        {
          "mcpServers": {
            "flashduty": {
              "command": "/path/to/flashduty-mcp-server",
              "args": ["stdio", "--app-key", "your_app_key_here"]
            }
          }
        }
        ```
        </CodeGroup>

        将 `/path/to/flashduty-mcp-server` 替换为二进制文件的实际路径。
      </Step>
      <Step title="验证连接">
        在 AI 工具中发送一条指令测试：

        ```
        请查询最近 24 小时内严重级别为 Critical 的故障
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>
