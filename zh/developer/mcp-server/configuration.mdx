Flashduty MCP Server 支持灵活的配置方式，您可以根据部署场景选择远程或本地配置方案。

## 配置优先级

当多种配置方式同时存在时，按以下优先级生效：

```
命令行参数 > 环境变量 > 配置文件 > 默认值
```

<Note>
远程服务仅支持通过 URL 参数配置，不支持配置文件和命令行参数。
</Note>

## 远程服务配置

使用远程服务 (`https://mcp.flashcat.cloud/mcp`) 时，通过 URL 查询参数进行配置：

```json
{
  "mcpServers": {
    "flashduty": {
      "url": "https://mcp.flashcat.cloud/mcp?toolsets=incidents,users&read_only=true",
      "headers": {
        "Authorization": "Bearer <your_flashduty_app_key>"
      }
    }
  }
}
```

**支持的 URL 参数**

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `toolsets` | 启用的工具集，逗号分隔 | `toolsets=incidents,users,channels` |
| `read_only` | 启用只读模式 | `read_only=true` |
| `base_url` | 自定义 API 地址 | `base_url=https://custom-api.example.com` |

**认证方式**

通过 HTTP 请求头 `Authorization` 传递 APP Key，格式为 `Bearer <your_app_key>`。

## 本地服务配置

### 环境变量

| 变量 | 说明 | 必填 | 默认值 |
| --- | --- | --- | --- |
| `FLASHDUTY_APP_KEY` | Flashduty APP Key | 是 | - |
| `FLASHDUTY_TOOLSETS` | 启用的工具集，逗号分隔 | 否 | 全部工具集 |
| `FLASHDUTY_READ_ONLY` | 只读模式（`1` 或 `true`） | 否 | `false` |
| `FLASHDUTY_OUTPUT_FORMAT` | 输出格式（`json` 或 `toon`） | 否 | `json` |
| `FLASHDUTY_BASE_URL` | Flashduty API 地址 | 否 | `https://api.flashcat.cloud` |
| `FLASHDUTY_LOG_FILE` | 日志文件路径 | 否 | stderr |
| `FLASHDUTY_ENABLE_COMMAND_LOGGING` | 记录请求/响应日志 | 否 | `false` |
| `TZ` | 日志时间戳时区 | 否 | 系统默认（容器中回退到 `Asia/Shanghai`） |

### 命令行参数

```bash
./flashduty-mcp-server stdio \
  --app-key your_app_key_here \
  --toolsets incidents,users,channels \
  --read-only \
  --output-format toon \
  --log-file /var/log/flashduty-mcp.log \
  --enable-command-logging
```

| 参数 | 说明 | 对应环境变量 |
| --- | --- | --- |
| `--app-key` | Flashduty APP Key | `FLASHDUTY_APP_KEY` |
| `--toolsets` | 启用的工具集，逗号分隔 | `FLASHDUTY_TOOLSETS` |
| `--read-only` | 启用只读模式 | `FLASHDUTY_READ_ONLY` |
| `--output-format` | 输出格式（`json` 或 `toon`） | `FLASHDUTY_OUTPUT_FORMAT` |
| `--base-url` | Flashduty API 地址 | `FLASHDUTY_BASE_URL` |
| `--log-file` | 日志文件路径 | `FLASHDUTY_LOG_FILE` |
| `--enable-command-logging` | 记录请求/响应日志 | `FLASHDUTY_ENABLE_COMMAND_LOGGING` |
| `--export-translations` | 导出翻译配置到 JSON 文件 | - |

## 工具集筛选

默认启用全部工具集。您可以通过 `toolsets` 参数仅启用所需的工具集，减少上下文大小，帮助 LLM 更精准地选择工具。

**可用工具集**

| 工具集 | 说明 | 工具数 | 包含写入工具 |
| --- | --- | --- | --- |
| `incidents` | 故障生命周期管理 | 8 | 是 |
| `changes` | 变更记录查询 | 1 | 否 |
| `status_page` | 状态页管理 | 4 | 是 |
| `users` | 成员和团队查询 | 2 | 否 |
| `channels` | 协作空间和分派策略 | 2 | 否 |
| `fields` | 自定义字段定义 | 1 | 否 |

使用 `all` 表示启用全部工具集。

<Tip>
如果您只需要查询功能，建议同时启用工具集筛选和只读模式，这样可以最小化工具列表，提升 AI 助手的响应准确性。
</Tip>

**配置示例**

<CodeGroup>
```bash 环境变量
export FLASHDUTY_TOOLSETS="incidents,users,channels"
```

```bash 命令行参数
./flashduty-mcp-server stdio --toolsets incidents,users,channels
```

```json 远程服务 URL 参数
{
  "url": "https://mcp.flashcat.cloud/mcp?toolsets=incidents,users,channels"
}
```
</CodeGroup>

## 只读模式

启用只读模式后，所有写入工具会被禁用，只保留查询类工具。适用于安全要求较高的场景。

被禁用的写入工具：
- `create_incident` - 创建故障
- `update_incident` - 更新故障
- `ack_incident` - 认领故障
- `close_incident` - 关闭故障
- `create_status_incident` - 创建状态页故障
- `create_change_timeline` - 添加变更时间线

<CodeGroup>
```bash 环境变量
export FLASHDUTY_READ_ONLY=true
```

```bash 命令行参数
./flashduty-mcp-server stdio --read-only
```

```json 远程服务 URL 参数
{
  "url": "https://mcp.flashcat.cloud/mcp?read_only=true"
}
```
</CodeGroup>

## 输出格式

MCP Server 支持两种输出格式：

### JSON（默认）

标准 JSON 格式，兼容性最好：

```json
{"members":[{"person_id":1,"person_name":"Alice"},{"person_id":2,"person_name":"Bob"}],"total":2}
```

### TOON（Token 优化）

[TOON (Token-Oriented Object Notation)](https://github.com/toon-format/toon) 是一种紧凑格式，可减少 30-50% 的 token 消耗，特别适合 LLM 场景：

```
members[2]{person_id,person_name}:
  1,Alice
  2,Bob
total: 2
```

<Tip>
TOON 格式对结构统一的对象数组（如成员列表、故障列表）效果最佳。主流 LLM 均可正确解析 TOON 格式。
</Tip>

<CodeGroup>
```bash 环境变量
export FLASHDUTY_OUTPUT_FORMAT=toon
```

```bash 命令行参数
./flashduty-mcp-server stdio --output-format toon
```
</CodeGroup>

## 自定义工具描述（仅本地）

您可以覆盖工具的默认描述，适用于国际化或团队定制场景。

### 配置文件方式

在二进制文件同目录下创建 `flashduty-mcp-server-config.json`：

```json
{
  "TOOL_CREATE_INCIDENT_DESCRIPTION": "创建一个新的故障工单",
  "TOOL_LIST_TEAMS_DESCRIPTION": "列出账户中的所有团队"
}
```

### 环境变量方式

使用 `FLASHDUTY_MCP_` 前缀加上描述键名：

```bash
export FLASHDUTY_MCP_TOOL_CREATE_INCIDENT_DESCRIPTION="创建一个新的故障工单"
```

<Note>
自定义工具描述功能仅在本地部署时可用，远程服务不支持。
</Note>

## 日志配置

### 日志文件

默认日志输出到 stderr。您可以通过 `--log-file` 或 `FLASHDUTY_LOG_FILE` 指定日志文件路径：

```bash
./flashduty-mcp-server stdio --log-file /var/log/flashduty-mcp.log
```

### 请求日志

启用 `--enable-command-logging` 后，服务会记录所有 MCP 请求和响应的详细日志，便于调试。

### 时区配置

通过 `TZ` 环境变量控制日志时间戳的时区：

```bash
# Docker 示例
docker run -i --rm \
  -e FLASHDUTY_APP_KEY=<your-app-key> \
  -e TZ=Asia/Shanghai \
  registry.flashcat.cloud/public/flashduty-mcp-server
```

常用时区值：`Asia/Shanghai`、`America/New_York`、`Europe/London`、`UTC`。

<Tip>
在没有时区数据的容器环境中（如 distroless 镜像），如果未设置 `TZ`，时区会自动回退到 `Asia/Shanghai`。
</Tip>

### 安全特性

MCP Server 内置了以下日志安全特性：

- **数据脱敏**：`APP_KEY` 和 `Authorization` 请求头等敏感信息在日志中自动掩码
- **日志截断**：过大的请求/响应体自动截断（默认 2KB），防止日志膨胀
- **链路追踪**：支持 W3C Trace Context 标准（`traceparent`），日志自动关联 `trace_id`，便于跨服务追踪

## Docker 完整配置示例

```bash
docker run -i --rm \
  -e FLASHDUTY_APP_KEY=<your-app-key> \
  -e FLASHDUTY_TOOLSETS="incidents,users,channels" \
  -e FLASHDUTY_READ_ONLY=1 \
  -e FLASHDUTY_OUTPUT_FORMAT=toon \
  -e TZ=Asia/Shanghai \
  registry.flashcat.cloud/public/flashduty-mcp-server
```

对应的 MCP 客户端配置：

```json
{
  "mcpServers": {
    "flashduty": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "FLASHDUTY_APP_KEY",
        "-e", "FLASHDUTY_TOOLSETS=incidents,users,channels",
        "-e", "FLASHDUTY_READ_ONLY=1",
        "-e", "FLASHDUTY_OUTPUT_FORMAT=toon",
        "-e", "TZ=Asia/Shanghai",
        "registry.flashcat.cloud/public/flashduty-mcp-server"
      ],
      "env": {
        "FLASHDUTY_APP_KEY": "your_flashduty_app_key"
      }
    }
  }
}
```
