Flashduty MCP Server 提供 6 大工具集、共 18 个工具。每个工具对应一个 Flashduty API 操作，AI 助手会根据您的自然语言指令自动选择合适的工具并填充参数。

<Note>
在只读模式下，所有写入工具（创建、更新、认领、关闭等）会被禁用。详见 [配置说明](/zh/developer/mcp-server/configuration)。
</Note>

## 故障管理 (incidents)

故障生命周期管理工具集，包含 8 个工具，覆盖故障的查询、创建、更新、认领、关闭和相似故障检索。

<Accordion title="query_incidents - 查询故障">
根据 ID、时间范围、状态、严重程度或协作空间查询故障。返回包含时间线、告警和响应人的完整信息。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_ids` | string | 否 | 逗号分隔的故障 ID，直接按 ID 查找。提供时忽略其他筛选条件。 |
| `progress` | string | 否 | 按状态筛选。可选值：`Triggered`（待处理）、`Processing`（处理中）、`Closed`（已关闭），多个用逗号分隔。 |
| `severity` | string | 否 | 按严重程度筛选。可选值：`Info`（轻微）、`Warning`（一般）、`Critical`（严重）。 |
| `channel_id` | number | 否 | 按协作空间 ID 筛选。 |
| `start_time` | number | 否 | 查询起始时间，Unix 时间戳（秒）。不指定 `incident_ids` 时必填。最大跨度 31 天。 |
| `end_time` | number | 否 | 查询结束时间，Unix 时间戳（秒）。不指定 `incident_ids` 时必填。 |
| `title` | string | 否 | 按故障标题关键词搜索。 |
| `limit` | number | 否 | 最大返回条数，范围 1-100，默认 20。 |
| `include_alerts` | boolean | 否 | 是否包含告警预览（前 20 条告警及总数），默认 `true`。 |

**使用示例**

```
查询最近 24 小时内所有严重级别为 Critical 的待处理故障
```
</Accordion>

<Accordion title="query_incident_timeline - 查询故障时间线">
查询故障的时间线事件，包括创建、分配、认领、关闭、通知、评论等操作记录。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_ids` | string | 是 | 逗号分隔的故障 ID。事件类型包括：`i_new`（创建）、`i_assign`（分配）、`i_ack`（认领）、`i_rslv`（关闭）、`i_notify`（通知）、`i_comm`（评论）、`i_r_*`（字段更新）。 |

**使用示例**

```
查看故障 INC-12345 的完整处理时间线
```
</Accordion>

<Accordion title="query_incident_alerts - 查询故障告警">
查询故障关联的告警信息，返回告警标题、严重程度、状态和标签。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_ids` | string | 是 | 逗号分隔的故障 ID。 |
| `limit` | number | 否 | 每个故障返回的最大告警数，范围 1-100，默认 20。 |

**使用示例**

```
列出故障 INC-12345 的所有关联告警
```
</Accordion>

<Accordion title="create_incident - 创建故障">
创建一个新的故障，指定标题和严重程度，可选关联协作空间或指定处理人员。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `title` | string | 是 | 故障标题，长度 3-200 字符。 |
| `severity` | string | 是 | 严重程度。可选值：`Info`（轻微）、`Warning`（一般）、`Critical`（严重）。 |
| `channel_id` | number | 否 | 关联的协作空间 ID。 |
| `description` | string | 否 | 故障描述，最大 6144 字符。 |
| `assigned_to` | string | 否 | 逗号分隔的成员 ID，指定为处理人员。可通过 `query_members` 查找成员 ID。 |

**使用示例**

```
创建一个 Critical 级别的故障，标题为"数据库主从延迟超过 30 秒"，描述为"MySQL 主从同步延迟持续增长"
```
</Accordion>

<Accordion title="update_incident - 更新故障">
更新故障的标题、描述、严重程度或自定义字段，只更新提供的字段。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_id` | string | 是 | 要更新的故障 ID。 |
| `title` | string | 否 | 新标题，长度 3-200 字符。 |
| `description` | string | 否 | 新描述，最大 6144 字符。 |
| `severity` | string | 否 | 新严重程度。可选值：`Info`、`Warning`、`Critical`。 |
| `custom_fields` | string | 否 | 自定义字段更新，JSON 格式。如 `{"field_name": "value"}`。字段名称格式：`^[a-z][a-z0-9_]*$`。可通过 `query_fields` 查询可用字段。 |

**使用示例**

```
将故障 INC-12345 的严重程度从 Warning 升级为 Critical，并更新描述为"影响范围扩大到全部用户"
```
</Accordion>

<Accordion title="ack_incident - 认领故障">
认领故障，将状态从「待处理」(Triggered) 变更为「处理中」(Processing)。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_ids` | string | 是 | 逗号分隔的故障 ID。认领操作会在时间线中记录操作人。 |

**使用示例**

```
认领故障 INC-12345 和 INC-12346
```
</Accordion>

<Accordion title="close_incident - 关闭故障">
关闭（解决）故障，将状态变更为「已关闭」(Closed)。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_ids` | string | 是 | 逗号分隔的故障 ID。关闭操作会在时间线中记录操作人。 |

**使用示例**

```
关闭故障 INC-12345
```
</Accordion>

<Accordion title="list_similar_incidents - 查找相似故障">
根据指定故障查找历史上的相似故障，辅助排查问题和回顾过往解决方案。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `incident_id` | string | 是 | 参考故障 ID，用于查找相似的历史故障。 |
| `limit` | number | 否 | 最大返回条数，范围 1-100，默认 20。 |

**使用示例**

```
查找与故障 INC-12345 相似的历史故障
```
</Accordion>

## 变更管理 (changes)

变更记录查询工具集，包含 1 个工具，用于查询部署、配置等变更记录，辅助故障与变更的关联分析。

<Accordion title="query_changes - 查询变更记录">
查询变更记录，支持按 ID、协作空间、时间范围和类型筛选。返回包含协作空间名称和创建者信息的富化数据。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `change_ids` | string | 否 | 逗号分隔的变更 ID，直接按 ID 查找。 |
| `channel_id` | number | 否 | 按协作空间 ID 筛选。 |
| `start_time` | number | 否 | 查询起始时间，Unix 时间戳（秒）。最大跨度 31 天。默认为 1 小时前。 |
| `end_time` | number | 否 | 查询结束时间，Unix 时间戳（秒）。默认为当前时间。 |
| `type` | string | 否 | 按变更类型筛选。 |
| `limit` | number | 否 | 最大返回条数，范围 1-100，默认 20。 |

**使用示例**

```
查询最近 1 小时内的所有变更记录
```
</Accordion>

## 状态页 (status_page)

状态页管理工具集，包含 4 个工具，用于查询状态页、查看变更事件、创建故障和更新时间线。

<Accordion title="query_status_pages - 查询状态页">
查询状态页配置信息，包含组件列表。支持按 ID 筛选或列出全部状态页。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page_ids` | string | 否 | 逗号分隔的状态页 ID。不提供时返回全部状态页。 |

**使用示例**

```
列出所有状态页及其组件信息
```
</Accordion>

<Accordion title="list_status_changes - 查询状态页变更事件">
查询状态页上的活跃故障或维护事件，返回尚未解决/完成的事件列表。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page_id` | number | 是 | 状态页 ID。 |
| `type` | string | 是 | 变更事件类型。可选值：`incident`（故障）、`maintenance`（维护）。 |

**使用示例**

```
查询状态页 123 上当前活跃的故障事件
```
</Accordion>

<Accordion title="create_status_incident - 创建状态页故障">
在状态页上创建故障事件，指定受影响的组件和状态更新。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page_id` | number | 是 | 状态页 ID。 |
| `title` | string | 是 | 故障标题，最大 255 字符。 |
| `message` | string | 否 | 初始更新消息，描述故障情况。 |
| `status` | string | 否 | 初始状态。可选值：`investigating`（调查中）、`identified`（已识别）、`monitoring`（监控中）、`resolved`（已解决）。默认 `investigating`。 |
| `affected_components` | string | 否 | 受影响组件，格式为 `组件ID:状态`，多个用逗号分隔。可用状态：`degraded`（性能降低）、`partial_outage`（部分中断）、`full_outage`（完全中断）。 |
| `notify_subscribers` | boolean | 否 | 是否通知状态页订阅者，默认 `true`。 |

**使用示例**

```
在状态页 123 上创建一个故障，标题为"API 响应延迟"，状态为"调查中"，受影响组件为 comp_001，状态为 degraded
```
</Accordion>

<Accordion title="create_change_timeline - 添加变更时间线">
向状态页的故障或维护事件添加时间线更新，可同时更新状态和受影响组件。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `page_id` | number | 是 | 状态页 ID。 |
| `change_id` | number | 是 | 变更事件 ID（故障或维护）。 |
| `message` | string | 是 | 更新消息，描述当前进展。 |
| `at` | number | 否 | 更新时间，Unix 时间戳（秒）。默认为当前时间。 |
| `status` | string | 否 | 新状态。故障可选值：`investigating`、`identified`、`monitoring`、`resolved`。维护可选值：`scheduled`、`ongoing`、`completed`。 |
| `component_changes` | string | 否 | 组件状态变更，JSON 数组格式。如 `[{"component_id":"xxx","status":"degraded"}]`。可用状态：`operational`、`degraded`、`partial_outage`、`full_outage`。 |

**使用示例**

```
更新状态页 123 上变更事件 456 的状态为 identified，并添加消息"已定位到数据库连接池耗尽问题"
```
</Accordion>

## 用户与团队 (users)

成员和团队查询工具集，包含 2 个工具。

<Accordion title="query_members - 查询成员">
按 ID、姓名或邮箱查询成员信息，返回包含团队归属的成员数据。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `person_ids` | string | 否 | 逗号分隔的成员 ID，直接按 ID 查找。 |
| `name` | string | 否 | 按姓名模糊搜索。 |
| `email` | string | 否 | 按邮箱搜索。 |

**使用示例**

```
查找名字包含"张"的所有成员
```
</Accordion>

<Accordion title="query_teams - 查询团队">
按 ID 或名称查询团队信息，返回包含成员详情的团队数据。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `team_ids` | string | 否 | 逗号分隔的团队 ID，直接按 ID 查找。 |
| `name` | string | 否 | 按团队名称搜索。 |

**使用示例**

```
查看运维团队的所有成员
```
</Accordion>

## 协作空间 (channels)

协作空间和分派策略查询工具集，包含 2 个工具。

<Accordion title="query_channels - 查询协作空间">
按 ID 或名称查询协作空间信息，返回包含团队和创建者名称的富化数据。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `channel_ids` | string | 否 | 逗号分隔的协作空间 ID，最大 1000 个。 |
| `name` | string | 否 | 按名称搜索（不区分大小写，子串匹配）。 |

**使用示例**

```
查询名称包含"生产环境"的协作空间
```
</Accordion>

<Accordion title="query_escalation_rules - 查询分派策略">
查询协作空间的分派策略，返回包含通知环节、通知对象（成员/团队/值班表）、Webhook、时间过滤和告警过滤条件的完整规则。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `channel_id` | number | 是 | 协作空间 ID。 |

**使用示例**

```
查看协作空间 100 的分派策略详情
```
</Accordion>

## 自定义字段 (fields)

自定义字段定义查询工具集，包含 1 个工具。

<Accordion title="query_fields - 查询自定义字段">
查询自定义字段定义信息，在更新故障的自定义字段前使用此工具了解可用字段。

**参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `field_ids` | string | 否 | 逗号分隔的字段 ID，直接按 ID 查找。 |
| `field_name` | string | 否 | 按精确字段名称搜索。字段名称格式：`^[a-z][a-z0-9_]*$`。 |

**使用示例**

```
列出所有可用的自定义字段及其定义
```
</Accordion>
