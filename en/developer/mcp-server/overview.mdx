## What is MCP

[Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction) is an open protocol that establishes standardized communication interfaces between AI applications and external tools/data sources. Through MCP, AI assistants can securely call external system APIs, retrieve real-time data, and perform operations without writing custom integration code for each system.

## What is Flashduty MCP Server

Flashduty MCP Server is a service built on the MCP protocol that provides seamless integration with the Flashduty API. It encapsulates Flashduty's core capabilities into a set of standardized tools, enabling AI assistants to directly manage incidents, query changes, operate status pages, and more.

### Core capabilities

Flashduty MCP Server provides **6 toolsets with 18 tools**, covering the complete incident management lifecycle:

| Toolset | Description | Tool count |
| --- | --- | --- |
| `incidents` | Incident lifecycle management (query, create, update, acknowledge, close, similar incident search) | 8 |
| `changes` | Change record queries (deployments, configuration changes, etc.) | 1 |
| `status_page` | Status page management (query, change events, create incidents, update timeline) | 4 |
| `users` | Member and team queries | 2 |
| `channels` | Channel and escalation rule queries | 2 |
| `fields` | Custom field definition queries | 1 |

### Use cases

- **Automated incident response**: AI assistants automatically identify and acknowledge incidents, find similar historical incidents to assist troubleshooting
- **Change correlation analysis**: Correlate incidents with recent change records to quickly identify root causes
- **Status page maintenance**: Create status page incidents and update progress through natural language instructions
- **Team collaboration queries**: Quickly find members, teams, and channel information
- **Data extraction and reporting**: Extract incident data from Flashduty to generate analysis reports

## Supported AI tools

Flashduty MCP Server is compatible with all AI tools and clients that support the MCP protocol:

| AI tool | Deployment method | Description |
| --- | --- | --- |
| [Claude Desktop](https://claude.ai) | Local (stdio) | Anthropic's official desktop client |
| [Cursor](https://www.cursor.com/) | Remote / Local | AI-powered code editor |
| [Windsurf](https://www.codeium.com/windsurf) | Local (stdio) | AI IDE by Codeium |
| [Claude Code](https://docs.anthropic.com/en/docs/claude-code) | Local (stdio) | Anthropic's command-line AI tool |
| Other MCP clients | Remote / Local | Any client that follows the MCP protocol can connect |

<Note>
Remote deployment requires the MCP client to support remote MCP server connections (Streamable HTTP transport). If your client does not support remote connections, use local deployment (stdio transport).
</Note>

## Architecture overview

Flashduty MCP Server supports two operating modes:

```
┌─────────────────────────────────────────────────────┐
│                 AI Tools / Clients                   │
│         (Claude Desktop, Cursor, Windsurf, etc.)    │
└──────────────┬──────────────────────┬────────────────┘
               │                      │
         Remote (HTTP)           Local (stdio)
               │                      │
               ▼                      ▼
┌──────────────────────┐  ┌──────────────────────────┐
│   Remote MCP Server  │  │    Local MCP Server       │
│ mcp.flashcat.cloud   │  │  Docker / Binary          │
└──────────┬───────────┘  └────────────┬─────────────┘
           │                           │
           └───────────┬───────────────┘
                       │
                       ▼
            ┌─────────────────────┐
            │   Flashduty API     │
            │ api.flashcat.cloud  │
            └─────────────────────┘
```

- **Remote mode**: Connect directly to Flashduty's hosted MCP service at `mcp.flashcat.cloud/mcp`, no installation required
- **Local mode**: Run MCP Server locally via Docker or binary, communicating with AI tools through stdio
