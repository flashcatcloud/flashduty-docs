title: "Quick start"
description: "Choose from remote service, Docker, or local binary to quickly connect Flashduty MCP Server and enable AI-driven incident management."
---

## Prerequisites

- A Flashduty account
- A Flashduty APP Key (create one in **Account Settings > APP Key**)
- An AI tool that supports the MCP protocol (such as Claude Desktop, Cursor, Windsurf, etc.)

<Warning>
The APP Key has full API access to your account. Keep it secure and do not share it with others or commit it to code repositories.
</Warning>

## Choose a deployment method

<Tabs>
  <Tab title="Remote service">
    Remote service is the simplest way to connect, requiring no component installation. It connects directly to Flashduty's hosted MCP service.

    **Applicable scenario**: Your MCP client supports remote MCP server connections (Streamable HTTP transport).

    <Steps>
      <Step title="Obtain an APP Key">
        Log in to the Flashduty console, go to **Account Settings > APP Key**, and create a new APP Key.
      </Step>
      <Step title="Configure the MCP client">
        Add the following configuration to your MCP client:

        <CodeGroup>
        ```json Cursor
        {
          "mcpServers": {
            "flashduty": {
              "url": "https://mcp.flashcat.cloud/mcp",
              "headers": {
                "Authorization": "Bearer <your_flashduty_app_key>"
              }
            }
          }
        }
        ```
        </CodeGroup>

        Replace `<your_flashduty_app_key>` with your actual APP Key.
      </Step>
      <Step title="Verify the connection">
        Send a command in your AI tool to verify that the MCP Server is working properly:

        ```
        Query all currently triggered incidents
        ```

        If configured correctly, the AI assistant will call the `query_incidents` tool and return the incident list.
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker">
    Run MCP Server locally via Docker, suitable for self-hosting or when the client does not support remote connections.

    **Prerequisite**: [Docker](https://www.docker.com/) is installed and running.

    <Steps>
      <Step title="Obtain an APP Key">
        Log in to the Flashduty console, go to **Account Settings > APP Key**, and create a new APP Key.
      </Step>
      <Step title="Configure the MCP client">
        Add the following configuration to your MCP client:

        ```json
        {
          "mcpServers": {
            "flashduty": {
              "command": "docker",
              "args": [
                "run",
                "-i",
                "--rm",
                "-e",
                "FLASHDUTY_APP_KEY",
                "registry.flashcat.cloud/public/flashduty-mcp-server"
              ],
              "env": {
                "FLASHDUTY_APP_KEY": "your_flashduty_app_key"
              }
            }
          }
        }
        ```

        The above configuration works with Cursor, Claude Desktop, and all MCP clients that support stdio transport.
      </Step>
      <Step title="Verify the connection">
        Send a command in your AI tool to test:

        ```
        List all team members
        ```

        If configured correctly, the AI assistant will call the `query_members` tool and return the member list.
      </Step>
    </Steps>

    <Tip>
    You can further configure the Docker container through environment variables, such as enabling read-only mode or specifying toolsets. See [Configuration](/en/developer/mcp-server/configuration) for details.
    </Tip>
  </Tab>

  <Tab title="Local binary">
    Download a pre-compiled binary or build from source to run MCP Server locally.

    <Steps>
      <Step title="Download the binary">
        Download the pre-compiled version for your operating system from [GitHub Releases](https://github.com/flashcatcloud/flashduty-mcp-server/releases).

        If you prefer to build from source:

        ```bash
        git clone https://github.com/flashcatcloud/flashduty-mcp-server.git
        cd flashduty-mcp-server
        go build -o flashduty-mcp-server ./cmd/flashduty-mcp-server
        ```
      </Step>
      <Step title="Obtain an APP Key">
        Log in to the Flashduty console, go to **Account Settings > APP Key**, and create a new APP Key.
      </Step>
      <Step title="Configure the MCP client">
        You can provide the APP Key through environment variables or command-line arguments:

        <CodeGroup>
        ```json Environment variable
        {
          "mcpServers": {
            "flashduty": {
              "command": "/path/to/flashduty-mcp-server",
              "args": ["stdio"],
              "env": {
                "FLASHDUTY_APP_KEY": "your_app_key_here"
              }
            }
          }
        }
        ```

        ```json Command-line argument
        {
          "mcpServers": {
            "flashduty": {
              "command": "/path/to/flashduty-mcp-server",
              "args": ["stdio", "--app-key", "your_app_key_here"]
            }
          }
        }
        ```
        </CodeGroup>

        Replace `/path/to/flashduty-mcp-server` with the actual path to the binary.
      </Step>
      <Step title="Verify the connection">
        Send a command in your AI tool to test:

        ```
        Query all Critical severity incidents in the last 24 hours
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>
