This page lists all 12 resources supported by the Flashduty Terraform Provider. Each resource includes complete attribute descriptions and HCL configuration examples.

<Tip>All resource names are prefixed with `flashduty_`. After creating a resource, the automatically generated `id` attribute can be used to reference it from other resources.</Tip>

<Accordion title="flashduty_team — Team" icon="users">

Manage Flashduty teams. Teams are the basic unit for organizing members and resources.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `team_name` | String | Team name, must be unique within the account |
| `description` | String | Team description |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique team identifier |

### Example

```hcl
resource "flashduty_team" "sre" {
  team_name   = "SRE Team"
  description = "Site Reliability Engineering team"
}

resource "flashduty_team" "payment" {
  team_name   = "Payment Service Team"
  description = "Team responsible for payment service operations"
}
```

</Accordion>

<Accordion title="flashduty_member_invite — Invite member" icon="user-plus">

Invite members to join a Flashduty account. Invited members can log in to activate their accounts after creation.

### Attributes

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `email` | String | Member email, at least one of `email` and `phone` is required |
| `phone` | String | Member phone number, at least one of `email` and `phone` is required |
| `member_name` | String | Member name, required when inviting by phone number |
| `country_code` | String | Phone country code, defaults to `CN` |
| `ref_id` | String | External reference ID |
| `role_ids` | List of Number | List of role IDs to assign to the member |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique invitation identifier |
| `member_id` | Number | Flashduty member ID assigned after invitation |

### Example

```hcl
# Invite by email and phone
resource "flashduty_member_invite" "john" {
  email        = "john.doe@example.com"
  member_name  = "John Doe"
  country_code = "CN"
  phone        = "10000000000"
}

# Invite by email only
resource "flashduty_member_invite" "alice" {
  email       = "alice@example.com"
  member_name = "Alice Smith"
}
```

</Accordion>

<Accordion title="flashduty_channel — Channel" icon="layer-group">

Manage Flashduty channels, with support for alert grouping and flapping detection configuration.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `channel_name` | String | Channel name |
| `team_id` | Number | ID of the owning team |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `description` | String | Channel description |
| `is_private` | Boolean | Whether the channel is private, defaults to `false` |
| `auto_resolve_timeout` | Number | Auto-resolve timeout in seconds, range 0-86400; not set means disabled |
| `auto_resolve_mode` | String | Auto-resolve mode: `trigger` (count from trigger) or `update` (count from update) |
| `disable_auto_close` | Boolean | Whether to disable automatic incident closing, defaults to `false` |
| `disable_outlier_detection` | Boolean | Whether to disable outlier incident detection, defaults to `false` |
| `managing_team_ids` | List of Number | Managing team ID list (up to 3) |
| `group` | Object | Alert grouping configuration, see below |
| `flapping` | Object | Flapping detection configuration, see below |

**`group` alert grouping configuration:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `method` | String | **Required**. Grouping method: `i` (intelligent alert grouping), `p` (pattern alert grouping), `n` (no grouping) |
| `time_window` | Number | Grouping time window in minutes, range 0-60; 0 means merge until incident closes |
| `i_keys` | List of String | Fields for intelligent grouping reference, such as `title`, `description`, `labels.service` |
| `i_score_threshold` | Number | Intelligent grouping similarity threshold, range 0.5-1.0, defaults to 0.9 |
| `equals` | List of List of String | Default dimensions for pattern grouping (AND groups within OR combinations) |
| `cases` | List of Object | Pattern grouping branch rules (up to 10), matched from top to bottom |
| `storm_thresholds` | List of Number | Alert storm warning thresholds (up to 5 values, each 2-10000) |
| `all_equals_required` | Boolean | Whether all grouping dimensions must be present, defaults to `false` |

**`flapping` detection configuration:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `is_disabled` | Boolean | **Required**. Whether to disable flapping detection |
| `max_changes` | Number | Maximum state changes within the time window, range 2-100, defaults to 4 |
| `in_mins` | Number | Statistics time window in minutes, range 1-1440, defaults to 60 |
| `mute_mins` | Number | Mute time window in minutes, range 0-1440, defaults to 120; 0 means no muting |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique channel identifier |

### Example

```hcl
resource "flashduty_team" "sre" {
  team_name   = "SRE Team"
  description = "Site Reliability Engineering team"
}

# Basic channel
resource "flashduty_channel" "basic" {
  channel_name = "Production Alerts"
  description  = "Channel for production environment alerts"
  team_id      = tonumber(flashduty_team.sre.id)
}

# Intelligent alert grouping
resource "flashduty_channel" "intelligent" {
  channel_name = "API Service Alerts"
  description  = "Alerts with intelligent grouping"
  team_id      = tonumber(flashduty_team.sre.id)

  group = {
    method            = "i"
    time_window       = 10
    i_score_threshold = 0.85
    i_keys            = ["title", "description", "labels.service"]
    storm_thresholds  = [100, 500]
  }
}

# Pattern alert grouping + flapping detection
resource "flashduty_channel" "rule_based" {
  channel_name = "Database Alerts"
  description  = "Database alerts with rule-based grouping"
  team_id      = tonumber(flashduty_team.sre.id)

  group = {
    method      = "p"
    time_window = 30
    equals      = [["title", "labels.host"]]

    cases = [
      {
        if = [
          {
            key  = "severity"
            oper = "IN"
            vals = ["Critical"]
          }
        ]
        equals = ["title", "labels.host", "labels.database"]
      }
    ]
  }

  flapping = {
    is_disabled = false
    max_changes = 5
    in_mins     = 60
    mute_mins   = 120
  }
}
```

</Accordion>

<Accordion title="flashduty_schedule — Schedule" icon="calendar">

Manage Flashduty schedules, with support for rotation rules, time restrictions, and notification configuration.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `schedule_name` | String | Schedule name |
| `description` | String | Schedule description |
| `layers` | List of Object | Schedule layers (rotation rules), see below |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `team_id` | Number | ID of the owning team |
| `status` | Number | Schedule status, 0 = enabled |
| `notify` | Object | Notification settings, see below |

**`layers` configuration:**

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `layer_name` | String | Yes | Layer name |
| `mode` | Number | Yes | Mode: 0 = regular, 1 = temporary |
| `layer_start` | Number | Yes | Layer start time (Unix timestamp) |
| `rotation_unit` | String | Yes | Rotation unit: `hour`, `day`, `week`, `month` |
| `rotation_value` | Number | Yes | Rotation interval value |
| `groups` | List of Object | Yes | On-call groups, containing `group_name` (optional) and `members` (required, with `person_ids` and optional `role_id`) |
| `layer_end` | Number | No | Layer end time (Unix timestamp) |
| `fair_rotation` | Boolean | No | Whether to enable fair rotation |
| `handoff_time` | Number | No | Handoff time in seconds |
| `restrict_mode` | Number | No | Restriction mode, 0 = no restriction |
| `restrict_periods` | List of Object | No | Time restriction periods, with `restrict_start` and `restrict_end` (seconds from midnight) |
| `day_mask` | Object | No | Weekday mask, `repeat` is a list of weekdays (0=Sunday, 1=Monday, ..., 6=Saturday) |
| `mask_continuous_enabled` | Boolean | No | Whether to enable continuous mask |

**`notify` configuration:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `advance_in_time` | Number | Advance notification time in seconds |
| `by` | Object | Notification channels, with `follow_preference` (Boolean) and `personal_channels` (e.g., `email`, `sms`, `phone`) |
| `fixed_time` | Object | Fixed-time notification, with `cycle` and `start` |
| `webhooks` | List of Object | Webhook notifications, with `type` and `settings` (JSON string) |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique schedule identifier |

### Example

```hcl
# Simple daily rotation schedule
resource "flashduty_schedule" "primary" {
  schedule_name = "Primary On-Call"
  description   = "Primary on-call rotation for the SRE team"

  layers = [
    {
      layer_name     = "Primary Layer"
      mode           = 0
      layer_start    = 1704038400
      rotation_unit  = "day"
      rotation_value = 1
      groups = [
        {
          group_name = "SRE Group"
          members = [
            {
              role_id    = 0
              person_ids = [75340551232454]
            }
          ]
        }
      ]
    }
  ]
}

# Schedule with business hours restriction and notifications
resource "flashduty_schedule" "business_hours" {
  schedule_name = "Business Hours Schedule"
  description   = "Weekday business hours on-call"

  layers = [
    {
      layer_name     = "Business Hours"
      mode           = 0
      layer_start    = 1704038400
      rotation_unit  = "week"
      rotation_value = 1
      fair_rotation  = true
      restrict_mode  = 1
      restrict_periods = [
        {
          restrict_start = 32400
          restrict_end   = 64800
        }
      ]
      day_mask = {
        repeat = [1, 2, 3, 4, 5]
      }
      groups = [
        {
          group_name = "Weekday Team"
          members = [
            {
              role_id    = 0
              person_ids = [75340551232454]
            }
          ]
        }
      ]
    }
  ]

  notify = {
    advance_in_time = 3600
    by = {
      follow_preference = true
      personal_channels = ["email"]
    }
  }
}
```

</Accordion>

<Accordion title="flashduty_incident — Incident" icon="triangle-exclamation">

Create and manage Flashduty incidents through code.

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `title` | String | Incident title |
| `incident_severity` | String | Incident severity: `Critical`, `Warning`, `Info` |

**Optional:**

| Attribute | Type | Description |

| `description` | String | Incident description, supports plain text or Markdown |

**Read-only:**

| Attribute | Type | Description |

| `id` | String | Unique incident identifier (ObjectID) |

```hcl
  title             = "API Latency Spike"

  incident_severity = "Warning"

}
```

<Accordion title="flashduty_escalate_rule — Escalation rule" icon="arrow-up-right-dots">

Manage escalation rules for channels, with support for multi-level layered notifications.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `channel_id` | Number | ID of the owning channel |
| `rule_name` | String | Rule name |
| `template_id` | String | Notification template ID |
| `layers` | List of Object | Escalation levels, see below |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `description` | String | Rule description |
| `aggr_window` | Number | Aggregation window in seconds, range 0-3600 |
| `priority` | Number | Rule priority |
| `filters` | List of Object | Alert matching filter conditions (OR between groups, AND between conditions) |
| `time_filters` | List of Object | Time filter conditions to control when the rule is active |

**`layers` escalation level configuration:**

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `target` | Object | Yes | Escalation target, see below |
| `escalate_window` | Number | No | Escalation wait time in minutes, defaults to 30 |
| `max_times` | Number | No | Maximum notification count, defaults to 2 |
| `notify_step` | Number | No | Notification interval in minutes, defaults to 10 |
| `force_escalate` | Boolean | No | Whether to escalate even after acknowledgement |

**`target` escalation target configuration:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `person_ids` | List of Number | Responder ID list |
| `team_ids` | List of Number | Team ID list |
| `schedule_ids` | List of Number | Schedule ID list |
| `emails` | List of String | Email address list |
| `by` | Object | Notification method configuration, with `follow_preference` (Boolean) and per-severity notification channels `critical`, `warning`, `info` (e.g., `email`, `sms`, `voice`, `im`) |
| `webhooks` | List of Object | Webhook notification targets, with `type` (e.g., `feishu_app`, `wecom`, `dingtalk`, `slack`) and `settings` (JSON string) |

**`filters` filter conditions:**

Each filter contains a `conditions` list (AND relationship), and filters are combined with OR logic. Each condition contains `key` (e.g., `severity`, `labels.xxx`), `oper` (`IN` or `NOTIN`), and `vals` (list of matching values).

**`time_filters` time filter conditions:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `start` | String | Start time (HH:MM format) |
| `end` | String | End time (HH:MM format) |
| `repeat` | List of Number | Active weekdays (0=Sunday, 1=Monday, ..., 6=Saturday) |
| `cal_id` | String | Calendar ID |
| `is_off` | Boolean | Whether to match the calendar's off-duty time |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique escalation rule identifier |

### Example

```hcl
# Basic escalation rule
resource "flashduty_escalate_rule" "default" {
  channel_id  = 6148622168454
  rule_name   = "Default Escalation"
  description = "Default escalation policy for all alerts"
  template_id = "6321aad26c12104586a88916"

  layers = [
    {
      escalate_window = 15
      target = {
        person_ids = [75340551232454]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
      }
    },
    {
      escalate_window = 30
      target = {
        schedule_ids = [4973305527804]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
      }
    }
  ]
}

# Escalation rule with filter conditions and time restrictions
resource "flashduty_escalate_rule" "critical" {
  channel_id  = 6148622168454
  rule_name   = "Critical Alerts Escalation"
  description = "Escalation for critical alerts during business hours"
  template_id = "6321aad26c12104586a88916"
  aggr_window = 60

  layers = [
    {
      max_times       = 3
      notify_step     = 10
      escalate_window = 15
      force_escalate  = true
      target = {
        person_ids = [75340551232454]
        by = {
          follow_preference = true
          critical          = ["email", "sms", "voice"]
          warning           = ["email", "sms"]
          info              = ["email"]
        }
        webhooks = [
          {
            type     = "wecom"
            settings = jsonencode({ token = "https://qyapi.weixin.qq.com/xxx" })
          }
        ]
      }
    }
  ]

  time_filters = [
    {
      start  = "09:00"
      end    = "18:00"
      repeat = [1, 2, 3, 4, 5]
    }
  ]

  filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Critical"]
        }
      ]
    }
  ]
}
```

</Accordion>

<Accordion title="flashduty_silence_rule — Silence rule" icon="volume-xmark">

Manage silence rules for channels to silence alert notifications during maintenance windows.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `channel_id` | Number | ID of the owning channel |
| `rule_name` | String | Rule name |
| `description` | String | Rule description |
| `filters` | List of Object | Filter conditions (OR between groups, AND between conditions), each condition contains `key`, `oper` (`IN`/`NOTIN`), `vals` |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `is_directly_discard` | Boolean | Whether to directly discard matching alerts |
| `priority` | Number | Rule priority |
| `time_filter` | Object | One-time time range, with `start_time` and `end_time` (Unix timestamps), mutually exclusive with `time_filters` |
| `time_filters` | List of Object | Recurring time conditions, with `start` (HH:MM), `end` (HH:MM), `repeat` (weekday list), mutually exclusive with `time_filter` |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique silence rule identifier |

### Example

```hcl
# Silence rule with fixed time window
resource "flashduty_silence_rule" "maintenance" {
  channel_id  = 6148622168454
  rule_name   = "Weekly Maintenance Window"
  description = "Silence alerts during weekly maintenance"

  filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Warning", "Info"]
        }
      ]
    }
  ]

  time_filter = {
    start_time = 1773121380
    end_time   = 1773207780
  }
}

# Recurring silence rule
resource "flashduty_silence_rule" "known_issue" {
  channel_id  = 6148622168454
  rule_name   = "Known Disk Space Alert"
  description = "Silence known disk space alerts from legacy server"

  filters = [
    {
      conditions = [
        {
          key  = "labels.host"
          oper = "IN"
          vals = ["legacy-server-01"]
        },
        {
          key  = "title"
          oper = "IN"
          vals = ["disk space"]
        }
      ]
    }
  ]

  time_filters = [
    {
      start  = "02:00"
      end    = "04:00"
      repeat = [0]
    }
  ]

  is_directly_discard = true
}
```

</Accordion>

<Accordion title="flashduty_inhibit_rule — Inhibit rule" icon="ban">

Manage inhibit rules for channels to suppress target alerts based on source alert conditions.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `channel_id` | Number | ID of the owning channel |
| `rule_name` | String | Rule name |
| `description` | String | Rule description |
| `source_filters` | List of Object | Source event filter conditions (OR between groups, AND between conditions) |
| `target_filters` | List of Object | Target event filter conditions (events to be suppressed) |
| `equals` | List of String | Fields that must be equal between source and target (e.g., `labels.host`) |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `is_directly_discard` | Boolean | Whether to directly discard matching alerts |
| `priority` | Number | Rule priority |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique inhibit rule identifier |

### Example

```hcl
# Critical severity suppresses Warning and Info
resource "flashduty_inhibit_rule" "critical_inhibits_warning" {
  channel_id  = 6148622168454
  rule_name   = "Critical Inhibits Warning"
  description = "Suppress warning alerts when critical alert is active for same host"

  source_filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Critical"]
        }
      ]
    }
  ]

  target_filters = [
    {
      conditions = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Warning", "Info"]
        }
      ]
    }
  ]

  equals = ["host", "service"]

  is_directly_discard = true
}

# Suppress downstream alerts when upstream service is down
resource "flashduty_inhibit_rule" "upstream_inhibits_downstream" {
  channel_id  = 6148622168454
  rule_name   = "Upstream Inhibits Downstream"
  description = "Suppress downstream alerts when upstream service is down"

  source_filters = [
    {
      conditions = [
        {
          key  = "labels.service"
          oper = "IN"
          vals = ["database", "cache"]
        }
      ]
    }
  ]

  target_filters = [
    {
      conditions = [
        {
          key  = "labels.service"
          oper = "IN"
          vals = ["api", "web"]
        }
      ]
    }
  ]

  equals = ["datacenter"]

  is_directly_discard = true
}
```

</Accordion>

<Accordion title="flashduty_field — Custom field" icon="tag">

Manage Flashduty custom fields to add structured metadata to incidents.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `field_name` | String | Field name, must match `^[a-zA-Z_][a-zA-Z0-9_]{0,39}$` |
| `display_name` | String | Field display name |
| `field_type` | String | Field type: `text`, `single_select`, `multi_select`, `checkbox` |
| `value_type` | String | Value type: `string`, `bool` |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `description` | String | Field description |
| `default_value` | String | Default value (JSON format) |
| `options` | String | Options (JSON array format), applicable to `single_select` and `multi_select` types |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique field identifier |
| `status` | String | Field status: `enabled`, `disabled`, `deleted` |
| `created_at` | Number | Creation timestamp |
| `updated_at` | Number | Update timestamp |

### Example

```hcl
# Single-select field
resource "flashduty_field" "priority" {
  field_name    = "priority"
  display_name  = "Priority Level"
  description   = "The priority level of the incident"
  field_type    = "single_select"
  value_type    = "string"
  options       = jsonencode(["P0", "P1", "P2", "P3"])

}

# Multi-select field
resource "flashduty_field" "affected_services" {
  field_name   = "affected_services"
  display_name = "Affected Services"
  description  = "Services impacted by this incident"
  field_type   = "multi_select"
  value_type   = "string"
  options      = jsonencode(["API", "Database", "Cache", "CDN", "Auth"])
}

# Text field
resource "flashduty_field" "customer_id" {
  field_name   = "customer_id"
  display_name = "Customer ID"
  description  = "The ID of the affected customer"
  field_type   = "text"
  value_type   = "string"
}

# Checkbox field
resource "flashduty_field" "requires_postmortem" {
  field_name    = "requires_postmortem"
  display_name  = "Requires Postmortem"
  description   = "Whether this incident requires a postmortem review"
  field_type    = "checkbox"
  value_type    = "bool"
  default_value = jsonencode(false)
}
```

<Accordion title="flashduty_route — Alert routing" icon="route">

Manage alert routing configuration for shared integrations. Each integration has only one routing configuration (upsert semantics), so only define one `flashduty_route` resource per `integration_id`.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `integration_id` | Number | Shared integration ID |

**Optional:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `cases` | List of Object | Conditional routing rules, see below |
| `default` | Object | Default routing configuration, with `channel_ids` (target channel ID list) |
| `sections` | List of Object | Routing rule groups, with `name` (must be unique) and `position` (0 means before the first rule) |

**`cases` routing rule configuration:**

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `if` | List of Object | Yes | Filter conditions (AND relationship), each with `key`, `oper` (`IN`/`NOTIN`), `vals` |
| `channel_ids` | List of Number | No | Target channel IDs (required in `standard` mode) |
| `routing_mode` | String | No | Routing mode: `standard` or `name_mapping` |

<Accordion title="flashduty_alert_pipeline — Alert pipeline" icon="filter">

Manage alert pipeline rules for integrations. Pipelines allow you to transform, drop, or inhibit alerts before they create incidents.

### Attributes

**Required:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `integration_id` | Number | ID of the owning integration |
| `rules` | List of Object | Pipeline rule list (executed from top to bottom), see below |

**`rules` pipeline rule configuration:**

| Attribute | Type | Required | Description |
|-----------|------|----------|-------------|
| `kind` | String | Yes | Rule type: `title_reset` (reset title), `description_reset` (reset description), `severity_reset` (reset severity), `alert_drop` (drop alert), `alert_inhibit` (inhibit alert) |
| `settings` | String | Yes | Rule settings (JSON format, use `jsonencode()`), content varies by kind |
| `if` | List of Object | No | Filter conditions (AND relationship), each with `key`, `oper` (`IN`/`NOTIN`), `vals` |

**`settings` content (by kind):**

| kind | settings content |
|------|------------------|
| `title_reset` | `{"title": "new title"}` |
| `description_reset` | `{"description": "new description"}` |
| `severity_reset` | `{"severity": "Critical"\|"Warning"\|"Info"}` |
| `alert_drop` | `{}` |
| `alert_inhibit` | `{"source_filters": [...], "equals": [...]}` |

**Read-only:**

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Resource identifier (same as integration_id) |

### Example

```hcl
resource "flashduty_alert_pipeline" "example" {
  integration_id = 5592304204454

  rules = [
    # Reset title for alerts from Prometheus
    {
      kind = "title_reset"
      if = [
        {
          key  = "labels.source"
          oper = "IN"
          vals = ["prometheus"]
        }
      ]
      settings = jsonencode({
        title = "[Prometheus] $${title}"
      })
    },

    # Upgrade production Info alerts to Warning
    {
      kind = "severity_reset"
      if = [
        {
          key  = "severity"
          oper = "IN"
          vals = ["Info"]
        },
        {
          key  = "labels.env"
          oper = "IN"
          vals = ["production"]
        }
      ]
      settings = jsonencode({
        severity = "Warning"
      })
    },

    # Drop known test alerts
    {
      kind = "alert_drop"
      if = [
        {
          key  = "title"
          oper = "IN"
          vals = ["test alert", "debug alert"]
        }
      ]
      settings = jsonencode({})
    },

    # Inhibit lower-severity alerts when Critical alert exists for the same host
    {
      kind = "alert_inhibit"
      settings = jsonencode({
        source_filters = [
          {
            key  = "severity"
            oper = "IN"
            vals = ["Critical"]
          }
        ]
        equals = ["labels.host", "labels.service"]
      })
    }
  ]
}
```

</Accordion>
