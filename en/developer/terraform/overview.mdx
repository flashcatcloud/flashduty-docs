## What is Terraform

[Terraform](https://www.terraform.io/) is an open-source Infrastructure as Code (IaC) tool by HashiCorp. It allows you to define and manage infrastructure resources through declarative configuration files, rather than manually operating in a console one by one. Terraform supports Providers for thousands of cloud services and SaaS platforms, and Flashduty is one of them.

## Flashduty Terraform Provider

The Flashduty Terraform Provider allows you to manage various resources on the Flashduty platform through Terraform configuration files. The Provider is published on the [Terraform Registry](https://registry.terraform.io/providers/flashcatcloud/flashduty/latest), and you can reference it directly.

### Supported resources

The Provider currently supports creating and managing **12 resource types**:

| Resource | Description |
|----------|-------------|
| `flashduty_team` | Team management |
| `flashduty_member_invite` | Invite members to the account |
| `flashduty_channel` | Channel management, with alert grouping and flapping detection |
| `flashduty_schedule` | Schedule management, with rotation rule configuration |
| `flashduty_incident` | Create and manage incidents through code |
| `flashduty_escalate_rule` | Escalation rules, with multi-level layered notifications |
| `flashduty_silence_rule` | Silence rules, to silence alerts during maintenance |
| `flashduty_inhibit_rule` | Inhibit rules, to suppress alerts based on conditions |
| `flashduty_field` | Custom fields, to add metadata to incidents |
| `flashduty_route` | Alert routing, to configure alert distribution rules for shared integrations |
| `flashduty_template` | Notification templates, to manage cross-channel notification templates |
| `flashduty_alert_pipeline` | Alert pipeline, to define alert transformation, drop, and inhibit rules |

### Supported data sources

The Provider also supports **13 data sources** for querying existing resources:

| Data source | Description |
|-------------|-------------|
| `flashduty_team` / `flashduty_teams` | Query teams |
| `flashduty_channel` / `flashduty_channels` | Query channels |
| `flashduty_member` / `flashduty_members` | Query members |
| `flashduty_field` / `flashduty_fields` | Query custom fields |
| `flashduty_template` / `flashduty_templates` | Query notification templates |
| `flashduty_route` / `flashduty_route_history` | Query routing rules and change history |
| `flashduty_alert_pipeline` | Query alert pipeline rules |

## When to use Terraform

<Tip>If your team is already using Terraform to manage other infrastructure, the Flashduty Provider lets you incorporate alert management configuration into the same IaC workflow.</Tip>

**Scenarios suited for Terraform:**

- **Multi-environment consistency** — You need to maintain identical alert configurations across development, staging, and production environments. Terraform ensures configuration consistency.
- **Version control and auditing** — All configuration changes are managed through Git, with every change recorded, supporting code review and rollback.
- **Bulk management** — When managing a large number of teams, channels, or escalation rules simultaneously, code-based management is far more efficient than manual operations.
- **Automation pipelines** — Integrate Flashduty configuration into CI/CD pipelines to automate infrastructure change deployments.
- **Cross-platform orchestration** — Orchestrate Flashduty resources alongside cloud resources, monitoring systems, and more, managing the entire infrastructure stack in a single configuration.

**Scenarios suited for the console UI:**

- You need to quickly create or modify a single resource.
- You need to view incident status and handling progress in real time.
- Team members are not familiar with Terraform or IaC tools.
