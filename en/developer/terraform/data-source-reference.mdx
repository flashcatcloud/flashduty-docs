This page lists all 13 data sources supported by the Flashduty Terraform Provider. Data sources are used to query existing Flashduty resources for reference in Terraform configurations.

<Tip>Data sources are declared with `data` blocks and their output values are referenced via `data.<type>.<name>.<attribute>`. Singular data sources query a single resource by ID, while plural data sources query resource lists.</Tip>

<Accordion title="flashduty_team — Query a single team" icon="user-group">

Query a single team by ID, name, or external reference ID.

### Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `team_id` | Number | Team ID, at least one of the three parameters is required |
| `team_name` | String | Team name |
| `ref_id` | String | External reference ID |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `team_name` | String | Team name |
| `description` | String | Team description |
| `created_at` | Number | Creation timestamp |
| `updated_at` | Number | Update timestamp |

### Example

```hcl
# Query by ID
data "flashduty_team" "by_id" {
  team_id = 12345
}

# Query by name
data "flashduty_team" "by_name" {
  team_name = "SRE Team"
}

# Query by external reference ID
data "flashduty_team" "by_ref" {
  ref_id = "sre-team-ref"
}

output "team_name" {
  value = data.flashduty_team.by_id.team_name
}
```

</Accordion>

<Accordion title="flashduty_teams — Query team list" icon="users">

Query multiple teams, with support for searching by name or description.

### Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | String | Search teams by name or description |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `teams` | List of Object | Team list, each containing `team_id`, `team_name`, `description` |

### Example

```hcl
# Query all teams
data "flashduty_teams" "all" {}

output "all_team_names" {
  value = [for team in data.flashduty_teams.all.teams : team.team_name]
}

# Search by name
data "flashduty_teams" "sre" {
  query = "SRE"
}

# Find a specific team using local
locals {
  sre_team = [for team in data.flashduty_teams.all.teams : team if team.team_name == "SRE Team"][0]
}

output "sre_team_id" {
  value = local.sre_team.team_id
}
```

</Accordion>

<Accordion title="flashduty_channel — Query a single channel" icon="layer-group">

Query detailed information for a single channel by ID.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `channel_id` | Number | Yes | Channel ID |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `channel_name` | String | Channel name |
| `description` | String | Channel description |
| `team_id` | Number | Owning team ID |
| `status` | String | Status |
| `is_private` | Boolean | Whether the channel is private |
| `auto_resolve_mode` | String | Auto-resolve mode |
| `auto_resolve_timeout` | Number | Auto-resolve timeout in seconds |
| `disable_auto_close` | Boolean | Whether auto-close is disabled |
| `disable_outlier_detection` | Boolean | Whether outlier incident detection is disabled |
| `managing_team_ids` | List of Number | Managing team ID list |
| `created_at` | Number | Creation timestamp |
| `updated_at` | Number | Update timestamp |

### Example

```hcl
data "flashduty_channel" "production" {
  channel_id = 6148622168454
}

output "channel_name" {
  value = data.flashduty_channel.production.channel_name
}

output "channel_team_id" {
  value = data.flashduty_channel.production.team_id
}
```

</Accordion>

<Accordion title="flashduty_channels — Query channel list" icon="layer-group">

### Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | String | Search by name or description |
| `team_ids` | List of Number | Filter by team IDs |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|

```hcl
# Query all channels
data "flashduty_channels" "all" {}

data "flashduty_channels" "sre_channels" {
  team_ids = [12345]
}

# Search by name
data "flashduty_channels" "production" {
  query = "production"
}
```

</Accordion>

<Accordion title="flashduty_members — Query member list" icon="users">

Query multiple members, with support for searching by name, email, or phone number.

### Query parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `query` | String | Search by name, email, or phone number |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `members` | List of Object | Member list, each containing `member_id`, `member_name`, `email`, `status` |

### Example

```hcl
# Query all members
data "flashduty_members" "all" {}

output "all_member_names" {
  value = [for m in data.flashduty_members.all.members : m.member_name]
}

data "flashduty_members" "search" {

  value = length(local.active_members)
}
```

<Accordion title="flashduty_field — Query a single custom field" icon="tag">

Query detailed information for a single custom field by ID.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | String | Yes | Field unique identifier |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `field_name` | String | Field name |
| `display_name` | String | Display name |
| `description` | String | Field description |
| `field_type` | String | Field type |
| `value_type` | String | Value type |
| `default_value` | String | Default value (JSON) |
| `options` | String | Options (JSON array) |
| `status` | String | Field status |
| `created_at` | Number | Creation timestamp |
| `updated_at` | Number | Update timestamp |

### Example

```hcl
data "flashduty_field" "priority" {
  id = "field-abc123"
}

output "field_name" {
  value = data.flashduty_field.priority.field_name
}

output "field_display_name" {
  value = data.flashduty_field.priority.display_name
}

output "field_options" {
  value = data.flashduty_field.priority.options != null ? jsondecode(data.flashduty_field.priority.options) : []
}
```

</Accordion>

<Accordion title="flashduty_fields — Query custom field list" icon="tags">

Query all custom fields.

### Query parameters

None.

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `fields` | List of Object | Field list, each containing `id`, `field_name`, `display_name`, `description`, `field_type`, `value_type`, `default_value`, `options`, `status`, `created_at`, `updated_at` |

### Example

```hcl
data "flashduty_fields" "all" {}

output "all_field_names" {
  value = [for f in data.flashduty_fields.all.fields : f.display_name]
}

# Find select-type fields
locals {
  select_fields = [for f in data.flashduty_fields.all.fields : f if f.field_type == "single_select" || f.field_type == "multi_select"]
}

output "select_field_count" {
  value = length(local.select_fields)
}
```

</Accordion>

<Accordion title="flashduty_route — Query alert routing" icon="route">

Query the current active alert routing configuration by integration ID.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `integration_id` | Number | Yes | Shared integration ID |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | String | Unique routing identifier |
| `status` | String | Routing status |
| `version` | Number | Routing configuration version number |
| `cases` | List of Object | Conditional routing rules, each containing `if` (filter conditions), `channel_ids`, `routing_mode`, `name_mapping_label`, `fallthrough` |
| `default` | Object | Default routing configuration, with `channel_ids` |
| `sections` | List of Object | Routing rule groups, with `name`, `position` |
| `created_at` | Number | Creation timestamp |
| `updated_at` | Number | Update timestamp |

### Example

```hcl
data "flashduty_route" "main" {
  integration_id = 5592304204454
}

output "route_version" {
  value = data.flashduty_route.main.version
}

output "route_status" {
  value = data.flashduty_route.main.status
}

output "default_channels" {
  value = data.flashduty_route.main.default != null ? data.flashduty_route.main.default.channel_ids : []
}

output "route_cases_count" {
  value = data.flashduty_route.main.cases != null ? length(data.flashduty_route.main.cases) : 0
}
```

</Accordion>

<Accordion title="flashduty_route_history — Query routing change history" icon="clock-rotate-left">

Query the alert routing change history for a shared integration.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `integration_id` | Number | Yes | Shared integration ID |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `items` | List of Object | History record list, each containing the following fields |

Each history record contains:

| Attribute | Type | Description |
|-----------|------|-------------|
| `version` | Number | Version number |
| `updated_at` | Number | Update timestamp |
| `updated_by` | Number | Updater user ID |
| `cases` | List of Object | Conditional routing rules for this version |
| `default` | Object | Default routing configuration for this version |
| `sections` | List of Object | Rule groups for this version |

### Example

```hcl
data "flashduty_route_history" "main" {
  integration_id = 5592304204454
}

output "history_count" {
  value = length(data.flashduty_route_history.main.items)
}

output "latest_version" {
  value = length(data.flashduty_route_history.main.items) > 0 ? data.flashduty_route_history.main.items[0].version : 0
}

output "all_versions" {
  value = [for item in data.flashduty_route_history.main.items : item.version]
}
```

</Accordion>

<Accordion title="flashduty_alert_pipeline — Query alert pipeline" icon="filter">

Query alert pipeline rules by integration ID.

### Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `integration_id` | Number | Yes | Integration ID |

### Output attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `rules` | List of Object | Pipeline rule list, each containing the following fields |

Each rule contains:

| Attribute | Type | Description |
|-----------|------|-------------|
| `kind` | String | Rule type: `title_reset`, `description_reset`, `severity_reset`, `alert_drop`, `alert_inhibit` |
| `settings` | String | Rule settings (JSON format) |
| `if` | List of Object | Filter conditions, each with `key`, `oper`, `vals` |

### Example

```hcl
data "flashduty_alert_pipeline" "example" {
  integration_id = 5592304204454
}

output "pipeline_rules" {
  value = data.flashduty_alert_pipeline.example.rules
}

output "rule_count" {
  value = length(data.flashduty_alert_pipeline.example.rules)
}
```

</Accordion>
