## Overview

The Event Center provides a timeline perspective to help you correlate incidents with change events across the time dimension. Through Gantt chart visualization, you can intuitively see which change operations occurred before and after an incident, helping you quickly determine whether the incident is related to a recent change.

## Gantt Chart Visualization

The core of the Event Center is the Gantt chart view, which simultaneously displays incidents and change events on the same timeline:

- **Incidents**: Displayed as time spans showing trigger and recovery periods, with colors distinguishing severity levels
- **Change Events**: Displayed in rows by change source (integration), with each source occupying its own row

The Gantt chart supports the following interactions:

| Action | Description |
| :--- | :--- |
| **Time Range Selection** | Adjust the viewing range using the time picker in the upper right corner; defaults to the last 6 hours |
| **Click Incident** | Click an incident bar on the Gantt chart to view the incident detail summary in a side panel |
| **Hover** | Hover over the timeline to display a reference line at the corresponding time point |

## Filtering

The Event Center provides a filter panel to help you focus on specific event ranges:

- **Label Filtering**: Filter by alert and change event labels, supporting multiple condition combinations
- **Change Source Filtering**: Select which change sources to display

Filter conditions can be passed via URL parameters, allowing you to share filtered result links with team members.

## Typical Use Cases

<AccordionGroup>
<Accordion title="Root cause analysis">
After an incident occurs, open the Event Center to check whether any change events happened around the incident trigger time. If you find that the incident trigger time closely aligns with a deployment change, that change is likely the root cause.
</Accordion>
<Accordion title="Change impact assessment">
After executing a major change, use the Event Center to monitor whether new incidents are triggered. If numerous incidents appear shortly after the change, the change may have introduced problems and rollback should be considered.
</Accordion>
</AccordionGroup>
