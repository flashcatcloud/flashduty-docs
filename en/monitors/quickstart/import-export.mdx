## Overview

Monitors supports importing and exporting alert rules, making it easy to migrate rule configurations between environments or back up existing rules.

## Export Rules

**Menu Entry**: Alert Rules → Export

Select the rules or rule groups to export as a JSON format file. The export includes complete rule configurations (query statements, threshold conditions, notification settings, etc.).

## Import Rules

**Menu Entry**: Alert Rules → Import

### Import from JSON

Upload a previously exported JSON format file. The system parses it and creates the corresponding alert rules. You need to specify the target rule group and associated data source during import.

### Import from Prometheus Alert Rules

Supports importing standard Prometheus AlertRules YAML format with `groups` as the root node. The system automatically converts Prometheus rules into Monitors alert rules.

```yaml
groups:
  - name: example
    rules:
      - alert: HighMemoryUsage
        expr: node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes < 0.1
        for: 5m
        labels:
          severity: critical
```

<Note>
YAML indentation must be correct, otherwise import will fail. After importing, it's recommended to review the rule configuration to confirm that data source associations and threshold conditions meet expectations.
</Note>

### Import from Rule Templates

You can select predefined templates from the rule template library to quickly create alert rules. See [Rule template library](/en/monitors/quickstart/rulesets) for details.
