## Overview

API keys are used for identity authentication and secure communication between `monitedge` alert engines and the Flashduty SaaS. Each engine must provide a valid API key during registration, which the SaaS uses to verify the engine's identity and establish a communication channel.

**Menu Entry**: Alert Engine → API Key Management

## Create an API Key

<Steps>
  <Step title="Go to API key management page">
    Under the Alert Engine menu, click API Key Management.
  </Step>
  <Step title="Create a new key">
    Click the create button, enter a key name (for identification purposes), and the system automatically generates a unique API key.
  </Step>
  <Step title="Save the key">
    The API key is only fully displayed once at creation time. Please save it securely.
  </Step>
</Steps>

## Manage API Keys

| Action | Description |
|--------|-------------|
| Rename | Change the display name of the API key without affecting the key value itself |
| Enable/Disable | Once disabled, engines using this key will be unable to communicate with the SaaS |
| Delete | Permanently delete the API key; associated engines need to switch to another valid key |

<Warning>
Disabling or deleting an API key that is currently in use will cause the corresponding alert engines to lose connectivity. Confirm that no engines are using the key before proceeding.
</Warning>
