The audit log records all member operations within your account, helping you track system changes, investigate anomalous actions, and meet security compliance requirements. You can flexibly search logs by time range, operator, event type, and other criteria.

## Viewing Audit Logs

---

**Configuration path**: Platform Management → Audit Log

Audit logs are displayed in a table, with each record containing the following information:

| Field | Description |
| --- | --- |
| **Time** | When the operation occurred, accurate to the second |
| **Username** | Name and avatar of the member who performed the operation |
| **Event Name** | The specific type of operation, such as creating a channel, modifying an escalation rule, etc. |
| **Event ID** | Unique request identifier for the operation, useful for troubleshooting |
| **Source IP** | Client IP address that initiated the operation |

Click on the time of a record to view the complete request details (JSON format) for that operation.

## Filtering and Searching

---

Audit logs provide multiple filtering criteria to help you quickly locate target operations:

| Filter | Description |
| --- | --- |
| **Time Range** | Select the query time interval; defaults to showing the last 7 days of logs. Each query can span up to 30 days; you can slide the window within your plan's data availability range |
| **User** | Filter by operator, supports searching by member name or email |
| **Event** | Filter by event type, supports multi-select and keyword search |
| **Event ID** | Enter an event ID to precisely match a specific operation record |

<Tip>
You can combine multiple filters. For example, you can specify both a time range and an operator to quickly find all operations by a specific member during a particular period.
</Tip>

**Data availability by plan:**

| Subscription Plan | Data Availability |
| --- | --- |
| Free | 30 days |
| Standard | 180 days |
| Professional | 366 days |

<Note>
Each query window is limited to 30 days. To view older data, adjust the start and end dates of your time range.
</Note>

## Pagination

---

Audit logs display 10 records per page. Use the pagination buttons at the bottom of the page to browse more records. The current page number and total pages are shown at the bottom.

<Note>
The audit log retention period depends on your subscription plan -- higher tiers retain logs longer. Contact support to upgrade if you need a longer retention period.
</Note>

## FAQ

---

<AccordionGroup>
  <Accordion title="Can audit logs be exported?">
    The console does not currently support direct export of audit logs. Please contact technical support if you need to export logs.
  </Accordion>
  <Accordion title="What operations are recorded in audit logs?">
    Audit logs record all write operations performed through the console or API within your account, including creating, modifying, and deleting resources, as well as member management and permission changes. Read-only operations (such as viewing pages) are not recorded.
  </Accordion>
</AccordionGroup>
